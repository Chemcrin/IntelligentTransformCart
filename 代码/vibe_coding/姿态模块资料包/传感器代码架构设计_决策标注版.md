## 1. 总体设计目标

JY61P 与 VL53L1X 组成小车底层感知模块，向运动控制层提供实时、稳定、带状态标志的数据。

| 模块 | 作用 | 主要输出 | 上层用途 |
|---|---|---|---|
| JY61P | 姿态与惯性测量 | 车体系 yaw、pitch、roll、角速度、加速度、温度、校准状态 | 直线纠偏、坡度判断、侧倾保护、震动/急停检测、姿态异常检测 |
| ATK-MS53L1M / VL53L1X | 车头前方距离检测 | 原始距离、滤波距离、近距离标志、超时/错误状态 | 避障、减速、停车、安全距离判断 |
| 统一传感器层 | 调度、状态聚合、快照缓存 | `SENSOR_Snapshot_t` | 电机控制、任务控制、调试上报 |

总体目标：

1. 传感器采集不阻塞主循环。
2. 任一 I2C 设备异常时，不永久拖死另一设备。
3. JY61P 累计误差通过代码侧校准与控制策略抑制。
4. VL53L1X 距离输出需要滤波、异常剔除和近距离优先保护。
5. 传感器层不直接控制电机，只向运动控制层提供可信数据和故障标志。

| `jy61p.c` | JY61P I2C 读写、解析、校准、滤波、状态机 | 寄存器读写、连续读取、物理量换算、车体系映射、累计误差抑制 | 使用传入的 `hi2c1` 指针 | 由 `sensor_update()` 周期调用 |
| `atk_ms53l1m.h` | VL53L1X/ATK 测距模块对外接口 | 地址宏、距离结构体、状态枚举、API 声明 | 声明需要 `I2C_HandleTypeDef *` | 被 `sensors.h` 或 `main.c` 引用 |

| 设备 | 7-bit 地址 | HAL 地址 | 冲突情况 |
|---|---:|---:|---|
| JY61P | `0x50` | `0xA0` | 无冲突 |

### 4.4 CubeIDE 配置要求

| 项目 | 配置 |
|---|---|
| I2C 初始化 | 由 CubeIDE 生成 `MX_I2C1_Init()`，传感器驱动不负责初始化外设。 |
| I2C 模式 | 7-bit 地址模式。 |
| I2C 速率 | 建议 100 kHz 起步；若稳定，可评估 400 kHz。 |
| PB6/PB7 | I2C 复用开漏。 |
| I2C 上拉 | 已确认上拉到 5V，软件不得主动输出高电平。 |
| DMA | 不启用。 |
| RTOS | 不使用。 |
| HAL 句柄 | `hi2c1`，通过初始化函数传给传感器模块。 |

### 4.5 I2C 5V 上拉注意事项

I2C 上拉到 5V 已由用户确认。软件侧必须遵守：

1. PB6/PB7 必须保持开漏或输入状态，不得配置为推挽输出高电平。
2. 总线恢复时，如果临时切 GPIO 模式，只允许输出低电平或释放总线，不允许主动输出高电平。
3. 传感器读取必须短超时，避免因 SDA 被拉低导致主循环长时间阻塞。
4. I2C 恢复完成后必须恢复 CubeIDE 生成的 I2C1 配置。

## 5. JY61P 架构设计

### 5.1 模块目标

JY61P 模块对上层提供车体系姿态与惯性数据：

| 输出 | 单位 | 用途 |
|---|---|---|
| `car_yaw_est_deg` | deg | 直线纠偏、转向判断。 |
| `car_pitch_deg` | deg | 坡度、车头上抬/下压判断。 |
| `car_roll_deg` | deg | 侧倾判断。 |
| `gyro_yaw_dps` | deg/s | 转向角速度辅助。 |
| `acc_forward_g` | g | 启动、急停、碰撞检测。 |
| `acc_right_g` | g | 侧向冲击检测。 |
| `acc_down_g` | g | 震动和重力方向判断。 |
| `temperature_c` | ℃ | 诊断。 |
| `imu_valid` | bool | 是否允许用于控制。 |
| `imu_calibrated` | bool | 是否完成零偏校准。 |

### 5.2 通信方式

| 项目 | 设计 |
|---|---|
| 外设 | I2C1。 |
| 引脚 | PB6/PB7。 |
| 7-bit 地址 | `0x50`。 |
| HAL 地址 | `0xA0`。 |
| 数据访问 | 主机主动读取寄存器。 |
| UART 模式 | 不使用。 |

### 5.3 初始化流程

| 步骤 | 处理 |
|---|---|
| 1 | `jy61p_init()` 接收 `I2C_HandleTypeDef *` 和当前时间戳。 |
| 2 | 保存 `hi2c1` 指针、地址、默认参数。 |
| 3 | 等待或确认模块上电稳定，建议系统上电后 1000 ms 内不强依赖数据。 |
| 4 | 使用短超时检测 `0x50` 是否应答。 |
| 5 | 读取 `0x34~0x40` 一组数据，确认不是全 0 或全 `0xFF`。 |
| 6 | 初始化滤波器、状态机、错误计数、校准缓存。 |
| 7 | 进入 `JY61P_STATE_RUNNING` 或 `JY61P_STATE_CALIBRATING`。 |

### 5.4 寄存器读取规划

推荐一次连续读取 `0x34~0x40` 共 13 个 16-bit 寄存器，即 26 字节。

| 数据 | 寄存器 | 字节数 | 换算 |
|---|---:|---:|---|
| AX | `0x34` | 2 | `raw / 32768 * 16g` |
| AY | `0x35` | 2 | `raw / 32768 * 16g` |
| AZ | `0x36` | 2 | `raw / 32768 * 16g` |
| GX | `0x37` | 2 | `raw / 32768 * 2000 °/s` |
| GY | `0x38` | 2 | `raw / 32768 * 2000 °/s` |
| GZ | `0x39` | 2 | `raw / 32768 * 2000 °/s` |
| Roll | `0x3D` | 2 | `raw / 32768 * 180°` |
| Pitch | `0x3E` | 2 | `raw / 32768 * 180°` |
| Yaw | `0x3F` | 2 | `raw / 32768 * 180°` |
| TEMP | `0x40` | 2 | `raw / 100 ℃` |

### 5.5 数据帧格式理解

| 数据来源 | 本项目是否使用 | 说明 |
|---|---|---|
| I2C 寄存器读取 | 使用 | 每个寄存器为 16 bit，低字节在前。没有 UART 帧头和 SUM 校验。 |
| UART 主动输出帧 | 不使用 | 不解析 `0x55` 帧头格式。 |

I2C 模式下数据有效性依靠：

1. HAL I2C 返回值。
2. 数据范围检查。
3. 连续样本跳变检查。
4. 时间戳超时检查。
5. 静止物理约束检查。

### 5.6 数据接收方式

| 方式 | 是否使用 | 原因 |
|---|---|---|
| I2C 轮询 | 使用 | 裸机主循环、数据量小、实现稳定。 |
| I2C 中断 | 不使用 | 当前需求不需要。 |
| I2C DMA | 不使用 | 用户明确不使用 DMA。 |
| UART 中断/DMA | 不使用 | JY61P 不走 UART。 |

### 5.7 车体系映射

已知安装方向：JY61P +Y 朝车前，+X 朝车右。

| 车体系字段 | 来源 | 标注 |
|---|---|---|
| `acc_forward_g` | JY61P `acc_y_g` | 已确认 |
| `acc_right_g` | JY61P `acc_x_g` | 已确认 |
| `acc_down_g` | 默认 `-acc_z_g` | 【已决策，需验证】 |
| `gyro_pitch_dps` | JY61P `gyro_x_dps` | 【已决策，需验证】 |
| `gyro_roll_dps` | JY61P `gyro_y_dps` | 【已决策，需验证】 |
| `gyro_yaw_dps` | JY61P `gyro_z_dps` | 符号需实测 |
| `car_pitch_deg` | JY61P `roll_deg` 加符号修正 | 【已决策，需验证】 |
| `car_roll_deg` | JY61P `pitch_deg` 加符号修正 | 【已决策，需验证】 |
| `car_yaw_deg` | JY61P `yaw_deg` 加符号修正 | 【已决策，需验证】 |

建议预留符号宏：

| 宏名 | 默认值 | 用途 |
|---|---:|---|
| `JY61P_SIGN_CAR_PITCH` | `+1` | 修正车体 pitch 方向。 |
| `JY61P_SIGN_CAR_ROLL` | `+1` | 修正车体 roll 方向。 |
| `JY61P_SIGN_CAR_YAW` | `+1` | 修正车体 yaw 方向。 |
| `JY61P_SIGN_ACC_DOWN` | `-1` | 修正 down 方向。 |

### 5.8 数据解析流程

| 步骤 | 处理 |
|---|---|
| 1 | 从 `0x34` 连续读取 26 字节。 |
| 2 | 每两个字节按低字节在前合成为 `int16_t`。 |
| 3 | 换算加速度、角速度、姿态角、温度。 |
| 4 | 扣除软件角速度零偏。 |
| 5 | 执行车体系映射和符号修正。 |
| 6 | 执行一阶低通滤波和 yaw wrap。 |
| 7 | 执行静止判据、跳变检查、范围检查。 |
| 8 | 数据有效才更新对外缓存和 `last_valid_ms`。 |
| 9 | 数据无效只更新错误计数，不覆盖最近有效快照。 |

### 5.9 数据有效性判断

| 检查项 | 建议规则 |
|---|---|
| I2C 返回值 | 必须为 `HAL_OK`。 |
| 原始数据 | 不应连续多次全 0 或全 `0xFF`。 |
| 加速度 | 单轴不超过 ±16 g；静止时模长接近 1 g。 |
| 角速度 | 单轴不超过 ±2000 °/s。 |
| 姿态角 | yaw/roll 在 -180~180，pitch 按实际输出范围检查。 |
| 时间戳 | 超过 `JY61P_DATA_TIMEOUT_MS` 未更新则 invalid。 |
| 跳变 | 静止状态下姿态角大跳变标记为可疑。 |

### 5.10 滤波策略

| 数据 | 滤波方式 | 说明 |
|---|---|---|
| Roll/Pitch | 一阶低通 | 抑制震动毛刺。 |
| Yaw | 互补滤波 + wrap | 结合 gyro 积分与 JY61P yaw。 |
| 加速度 | 原始值 + 轻低通双输出 | 原始值用于急停/冲击，滤波值用于趋势判断。 |
| 角速度 | 扣零偏后一阶低通 | 用于 yaw 估计与运动状态判断。 |
| 温度 | 慢速更新 | 仅诊断，不参与闭环。 |

### 5.11 累计误差抑制方案

本项目不使用卡尔曼滤波。JY61P 累计误差通过组合策略抑制。

#### 5.11.1 上电静止零偏校准

| 项目 | 设计 |
|---|---|
| 触发条件 | 系统上电、电机未使能、小车静止。 |
| 采样周期 | 10 ms。 |
| 采样次数 | 200 次，约 2 s。 |
| 采样内容 | 三轴 gyro、三轴 acc、yaw。 |
| 静止判据 | 三轴角速度小于阈值，加速度模长在 0.95~1.05 g，姿态变化小。 |
| 输出 | `gyro_bias_x/y/z_dps`、`yaw_zero_deg`、`imu_calibrated`。 |
| 保存 | 默认 RAM 保存，不频繁写 Flash。 |

#### 5.11.2 运行中静止零速更新

| 条件 | 建议值 |
|---|---:|
| 电机目标速度 | 0 |
| 静止持续时间 | ≥ 800 ms |
| 三轴角速度 | < 1.5 °/s |
| 加速度模长 | 0.95~1.05 g |

零速更新只缓慢修正 gyro 零偏，不频繁改 `yaw_zero_deg`，防止控制中 yaw 突跳。

#### 5.11.3 分段航向归零

| 场景 | 处理 |
|---|---|
| 每段直线开始 | 记录当前 `car_yaw_est_deg` 为 `yaw_target_deg`。 |
| 直线行驶中 | 使用 `wrap(yaw_target_deg - car_yaw_est_deg)` 作为纠偏误差。 |
| 到达目标点并静止 | 允许重置段内 yaw 参考。 |
| 转弯完成后 | 建立下一段直线的 yaw 目标。 |

#### 5.11.4 互补滤波

| 参数 | 建议值 |
|---|---:|
| gyro 积分权重 | 0.95~0.98 |
| JY61P yaw 修正权重 | 0.02~0.05 |
| yaw 控制误差限幅 | ±30° |
| 静止漂移报警 | 5 分钟超过 5°，需实测调整 |

概念流程：

1. 用扣零偏后的 `gyro_yaw_dps` 积分得到短期 yaw。
2. 用 JY61P 原始 yaw 作为低频参考。
3. 每次融合前对角度差做 wrap 到 -180~180。
4. 数据无效时冻结或降低 yaw 控制权重。
5. 数据超时时禁用 yaw 闭环。

### 5.12 与运动控制接口

| 传感器层提供 | 运动控制层使用 |
|---|---|
| `car_yaw_est_deg` | 直线纠偏。 |
| `car_pitch_deg` | 坡度限速。 |
| `car_roll_deg` | 侧倾保护。 |
| `acc_forward_g` | 急停、碰撞、启动检测。 |
| `gyro_yaw_dps` | 转向角速度辅助。 |
| `imu_valid`、`imu_calibrated` | 是否启用姿态闭环。 |

## 6. ATK-MS53L1M / VL53L1X 架构设计

### 6.1 模块目标

| 输出 | 单位 | 用途 |
|---|---|---|
| `raw_distance_mm` | mm | 原始测距，调试和滤波输入。 |
| `filtered_distance_mm` | mm | 避障与减速控制。 |
| `is_warning` | bool | 进入预警距离。 |
| `is_too_close` | bool | 进入硬停车距离。 |
| `range_valid` | bool | 距离是否可信。 |
| `last_valid_ms` | ms | 判断数据是否过期。 |
| `state` | enum | 初始化、运行、错误、超时状态。 |

### 6.2 通信与地址

| 项目 | 设计 |
|---|---|
| 实际芯片 | VL53L1X 直连。 |
| 模块命名 | 仍使用 `atk_ms53l1m.*` 和 `ms53l1m_` 前缀，方便对应硬件模块名。 |
| 外设 | I2C1。 |
| 引脚 | PB6/PB7。 |
| 7-bit 地址 | `0x29`。 |
| HAL 地址 | `0x52`。 |

### 6.3 初始化流程

| 步骤 | 处理 |
|---|---|
| 1 | `ms53l1m_init()` 接收 `I2C_HandleTypeDef *` 和当前时间戳。 |
| 2 | 保存地址 `0x29` 和 HAL 地址 `0x52`。 |
| 3 | 使用短超时检测设备 ready。 |
| 4 | 调用 VL53L1X 默认启动封装。 |
| 5 | 清空滤波缓存、错误计数、距离缓存。 |
| 6 | 进入 `MS53L1M_STATE_RUNNING`。 |

> VL53L1X 的 ID、Long 模式、Timing Budget、Inter-measurement 等细节按用户要求忽略，后续实现时由选定的底层 VL53L1X 库或默认启动函数内部处理。

### 6.4 测距模式选择

| 模式 | 是否使用 | 说明 |
|---|---|---|
| 连续测量 | 使用 | 适合小车实时避障。 |
| 单次测量 | 不作为默认 | 可作为调试接口保留。 |
| UART 输出 | 不使用 | 本项目 VL53L1X 直连 I2C。 |

### 6.5 测距周期和有效范围

| 参数 | 默认值 | 说明 |
|---|---:|---|
| 轮询周期 | 20 ms | 主循环查询是否有新距离。 |
| 数据超时 | 300~500 ms | 超过后禁止高速前进。 |
| 最小有效距离 | 40 mm | 小于该值通常不作为普通有效值，但可触发近距离异常。 |
| 最大有效距离 | 4000 mm | 4 m 级测距上限。 |
| 预警距离 | 600 mm | 限速或提醒。 |
| 减速距离 | 300 mm | 明显减速。 |
| 停车距离 | 150 mm | 请求电机立即停车。 |

### 6.6 数据读取流程

| 步骤 | 处理 |
|---|---|
| 1 | 检查状态是否为 `RUNNING`。 |
| 2 | 到达 20 ms 轮询周期后读取距离或查询 ready。 |
| 3 | 如果无新数据，立即返回，不阻塞等待。 |
| 4 | 如果有新数据，读取 raw distance。 |
| 5 | 执行范围检查和状态检查。 |
| 6 | 有效样本进入中值滤波。 |
| 7 | 中值结果进入滑动平均。 |
| 8 | 更新 `filtered_distance_mm`、`last_valid_ms`、标志位。 |
| 9 | 异常样本只更新错误计数，不覆盖稳定距离。 |

### 6.7 滤波与异常剔除

| 层级 | 方法 | 默认值 | 说明 |
|---|---|---:|---|
| 第一层 | 范围检查 | 40~4000 mm | 超出范围不进入普通滤波。 |
| 第二层 | 中值滤波 | 3 点 | 抑制单点毛刺。 |
| 第三层 | 滑动平均 | 5 点 | 平滑稳定距离。 |
| 第四层 | 跳变剔除 | 300 mm | 跳变过大时要求连续确认。 |
| 安全例外 | 近距离优先 | 2 次确认 | 连续两次低于停车阈值则立即置位 `is_too_close`。 |

### 6.8 单车头激光限制

| 限制 | 处理 |
|---|---|
| 只能检测正前方 | 控制层不得把它当成全向避障。 |
| 斜面、黑色、透明目标可能异常 | 用滤波与无效状态处理。 |
| 快速靠近障碍 | 近距离样本优先触发停车，不被平均滤波延迟。 |
| 数据失效 | 限速或停车。 |

## 7. 持续测量与实时更新机制

### 7.1 初始化阶段

| 顺序 | 操作 |
|---|---|
| 1 | `HAL_Init()`。 |
| 2 | 系统时钟初始化。 |
| 3 | GPIO、I2C1、UART1 等 CubeIDE 生成初始化。 |
| 4 | 调用 `sensor_init_all(&hi2c1)`。 |
| 5 | JY61P 检测地址 `0x50`，VL53L1X 检测地址 `0x29`。 |
| 6 | 启动 VL53L1X 默认连续测距。 |
| 7 | 启动或执行 JY61P 上电静止校准。 |
| 8 | 初始化统一快照和故障标志。 |

### 7.2 主循环调度

| 周期 | 任务 |
|---|---|
| 每轮循环 | 获取 `HAL_GetTick()`。 |
| 10 ms | `jy61p_update()`。 |
| 20 ms | `ms53l1m_update()`。 |
| 20 ms | `sensor_update_snapshot()`。 |
| 10~20 ms | 电机控制层读取快照，计算 UART1 电机命令。 |
| 500~1000 ms | 输出诊断日志。 |

### 7.3 数据缓存方式

| 缓存 | 说明 |
|---|---|
| JY61P 内部缓存 | 原始数据、滤波数据、校准参数、状态。 |
| VL53L1X 内部缓存 | 原始距离、滤波窗口、稳定距离、状态。 |
| 统一快照 | 将两个模块最近有效数据复制为 `SENSOR_Snapshot_t`。 |
| 旧数据策略 | 旧数据可用于显示和诊断；超时后 `valid=false`，不得用于闭环控制。 |

### 7.4 避免阻塞主循环

1. 不在 update 函数中使用长 `HAL_Delay()`。
2. I2C 读写使用 5~10 ms 短超时。
3. VL53L1X 未 ready 时直接返回。
4. JY61P 校准分步执行，不一次阻塞 2 秒。
5. 总线恢复有限次数执行，不无限重试。

### 7.5 I2C 总线恢复

建议统一传感器层提供：

| 接口 | 作用 |
|---|---|
| `sensor_i2c_check_bus()` | 检查 bus busy、SDA/SCL 状态。 |
| `sensor_i2c_recover_bus()` | 释放 I2C 总线，必要时 SCL 产生有限脉冲。 |
| `sensor_try_recover_device()` | 对单个设备按间隔重新检测。 |
| `sensor_mark_device_fault()` | 标记单设备故障，避免错误扩散。 |

总线恢复注意事项：

1. 因 I2C 上拉为 5V，恢复 GPIO 不能推挽输出高电平。
2. 恢复完成后重新进入 I2C 外设模式。
3. 恢复时不要无限阻塞主循环。
4. 一个设备错误时，先标记单设备 fault，再判断总线是否整体异常。

## 9. 状态机与错误处理

### 9.1 通用状态

| 状态 | 含义 |
|---|---|
| `UNINIT` | 未初始化。 |
| `INITING` | 初始化中。 |
| `RUNNING` | 正常运行。 |
| `CALIBRATING` | 校准中，主要用于 JY61P。 |
| `DATA_TIMEOUT` | 数据超时。 |
| `COMM_ERROR` | I2C 通信错误。 |
| `DATA_INVALID` | 数据不可信。 |
| `NEED_REINIT` | 需要重新初始化。 |
| `DISABLED` | 被上层禁用。 |

### 9.2 JY61P 状态机

| 状态 | 进入条件 | 退出条件 | 上层处理 | 是否允许旧数据 |
|---|---|---|---|---|
| `UNINIT` | 上电默认 | 调用初始化 | 不使用 | 否 |
| `INITING` | 初始化开始 | 成功或失败 | 等待 | 否 |
| `CALIBRATING` | 上电或命令触发校准 | 校准成功/失败 | 禁止高速运动 | 可显示，不建议闭环 |
| `RUNNING` | 初始化成功且数据有效 | 超时/错误/校准 | 正常使用 | 是 |
| `DATA_TIMEOUT` | 超时未更新 | 新有效数据恢复 | 禁用 yaw 闭环 | 只诊断 |
| `COMM_ERROR` | 连续 I2C 失败 | 恢复成功 | 降级或停车 | 只诊断 |
| `DATA_INVALID` | 数据范围或物理约束失败 | 连续有效样本恢复 | 不用于控制 | 否 |
| `NEED_REINIT` | 错误累计超阈值 | 重新初始化成功 | 触发恢复 | 否 |

### 9.3 VL53L1X 状态机

| 状态 | 进入条件 | 退出条件 | 上层处理 | 是否允许旧数据 |
|---|---|---|---|---|
| `UNINIT` | 上电默认 | 调用初始化 | 不使用 | 否 |
| `INITING` | 初始化开始 | 成功或失败 | 等待 | 否 |
| `RUNNING` | 初始化成功且距离有效 | 超时/错误 | 正常避障 | 是 |
| `DATA_TIMEOUT` | 长时间无新距离 | 新有效距离恢复 | 限速或停车 | 只短时显示 |
| `COMM_ERROR` | 连续 I2C 错误 | 恢复成功 | 禁止高速前进 | 否 |
| `DATA_INVALID` | 距离异常 | 连续有效样本恢复 | 不使用该距离 | 否 |
| `NEED_REINIT` | 错误超阈值 | 重新初始化成功 | 触发恢复 | 否 |

### 9.4 错误处理原则

| 错误等级 | 示例 | 处理 |
|---|---|---|
| 轻微 | 单次 I2C 失败、单次距离毛刺 | 错误计数加一，保留旧值。 |
| 中等 | 连续数次失败、短时超时 | 标记 invalid，控制层降级。 |
| 严重 | 总线卡死、长时间无数据 | 停车或禁止自动运行，触发总线恢复。 |
| 恢复 | 连续有效样本达到阈值 | 清除连续错误，恢复 running。 |

### 13.2 CubeIDE 生成代码注意事项

| 文件 | 处理方式 |

### 13.3 `main.c` 调用关系

初始化阶段：

1. HAL 初始化。
2. 系统时钟初始化。
3. GPIO 初始化。
4. I2C1 初始化。
5. UART1 初始化。
6. 调用 `sensor_init_all(&hi2c1)`。
7. 进入主循环。

主循环阶段：

1. 获取 `now_ms = HAL_GetTick()`。
2. 调用 `sensor_update(now_ms)`。
3. 电机控制层调用 `sensor_get_snapshot()`。
4. 电机控制层通过 UART1 输出控制命令。
5. 定期输出诊断日志。

### 14.1 编译层面

| 编号 | 验收标准 |
|---|---|
| C-01 | 工程在 STM32CubeIDE 1.19.0 中可编译通过。 |

| C-05 | 不启用 RTOS，不依赖 DMA。 |

### 14.2 初始化层面

| 编号 | 验收标准 |
|---|---|
| I-01 | I2C1 使用 PB6/PB7。 |
| I-02 | I2C 扫描或 ready 检测能发现 JY61P `0x50`。 |
| I-03 | I2C 扫描或 ready 检测能发现 VL53L1X `0x29`。 |
| I-04 | 初始化失败时系统不死循环，能进入错误状态。 |
| I-05 | PB6/PB7 不被配置为推挽高电平输出。 |

### 14.3 JY61P 数据读取层面

| 编号 | 验收标准 |
|---|---|
| J-01 | 静止水平放置时，姿态角输出稳定。 |
| J-02 | 手动旋转车体时 `car_yaw_est_deg` 连续变化。 |
| J-03 | 抬高车头时 `car_pitch_deg` 方向正确；若相反，通过符号宏修正。 |
| J-04 | 车体侧倾时 `car_roll_deg` 方向正确；若相反，通过符号宏修正。 |
| J-05 | 静止时扣零偏后的角速度接近 0。 |
| J-06 | 超过数据超时阈值未更新时，`imu_valid=false`。 |
| J-07 | I2C 读取失败不覆盖最近有效姿态数据。 |

### 14.4 JY61P 校准与误差抑制层面

| 编号 | 验收标准 |
|---|---|
| JC-01 | 上电静止约 2 s 后生成 gyro 零偏。 |
| JC-02 | 校准完成后 `imu_calibrated=true`。 |
| JC-03 | 运动中触发校准应被拒绝或延迟。 |
| JC-04 | 静止运行 5 min，yaw 漂移被记录，零速更新能降低后续漂移趋势。 |
| JC-05 | 每段直线开始能建立新的 `yaw_target_deg`。 |
| JC-06 | IMU 超时后电机控制层禁用 yaw 闭环。 |

### 14.5 VL53L1X 测距层面

| 编号 | 验收标准 |
|---|---|
| L-01 | VL53L1X 地址固定使用 `0x29`。 |
| L-02 | 距离单位统一为 mm。 |
| L-03 | 40~4000 mm 范围内距离变化能被读取。 |
| L-04 | 单次毛刺不会污染 `filtered_distance_mm`。 |
| L-05 | 距离低于 600 mm 时 `is_warning=true`。 |
| L-06 | 距离低于 150 mm 且连续确认时 `is_too_close=true`。 |
| L-07 | 距离数据超时后 `laser_ok=false`。 |

### 14.6 I2C 总线与连续运行

| 编号 | 验收标准 |
|---|---|
| B-01 | JY61P 与 VL53L1X 同时运行 30 min 不死机。 |
| B-02 | 任一设备断开后，系统能标记对应设备错误。 |
| B-03 | 任一设备异常后，另一个设备不应被永久拖死。 |
| B-04 | 总线异常后，统一传感器层可尝试 bus recovery。 |
| B-05 | 主循环无明显卡顿，UART1 电机控制仍能按周期运行。 |

### 14.7 与电机控制接口层面

| 编号 | 验收标准 |
|---|---|
| M-01 | 电机控制层能读取 `SENSOR_Snapshot_t`。 |
| M-02 | 激光 `is_too_close=true` 时，电机控制层可通过 UART1 请求停车。 |
| M-03 | JY61P 无效时，电机控制层禁用航向闭环。 |
| M-04 | VL53L1X 无效时，电机控制层禁止高速前进。 |
| M-05 | 传感器层不直接发送电机命令。 |

## 16. 未验证风险

| 风险项 | 当前状态 | 影响 | 建议处理 |
|---|---|---|---|
| STM32C8T6 具体完整型号 | 用户给出 STM32C8T6，本文按 STM32F103C8T6 类资源处理 | 若具体芯片资源差异较大，外设或 Flash/RAM 可能不同 | 【已决策，需验证】实物丝印和 `.ioc` 最终确认。 |
| JY61P Z 轴方向 | 用户明确 X 朝右、Y 朝前；Z 未明确 | 可能导致 down、yaw、roll/pitch 符号不一致 | 【已决策，需验证】默认板面朝上，保留符号宏，实测后修正。 |
| JY61P 姿态角映射 | 安装方向导致 JY61P 原始 Roll/Pitch 与车体系不完全同名 | 上层若直接使用原始角度可能控制反向 | 【已决策，需验证】统一输出 `car_*` 字段，禁止上层直接使用原始角。 |
| JY61P yaw 累计误差 | 纯 IMU 航向无法完全消除漂移 | 长时间直线控制可能偏航 | 使用上电校准、零速更新、分段归零、互补滤波、误差限幅。 |
| 实车震动 | 未完成实车验证 | 姿态和角速度抖动 | 实测调整滤波系数、静止阈值、yaw 限幅。 |
| I2C 上拉 5V | 已确认 | GPIO 恢复若主动输出高电平可能有硬件风险 | 保持开漏；恢复过程只拉低或释放总线。 |
| I2C 共总线异常 | 两传感器共用 PB6/PB7 | 一个设备异常可能影响另一个设备 | 短超时、错误隔离、bus recovery、断开单设备验收测试。 |
| VL53L1X 默认启动封装 | 用户要求忽略底层细节，但后续仍需具体库实现 | 后续写代码时需要绑定 ST API、ATK 示例库或自写底层读写 | 【已决策，需验证】架构只暴露 `ms53l1m_start_default()`，实现阶段绑定具体库。 |
| VL53L1X 有效距离表现 | 按 40~4000 mm 设计 | 黑色、透明、斜面、强光可能异常 | 实测 100/150/300/600/1000/2000/4000 mm，调整阈值。 |
| 单个车头激光视野有限 | 已知限制 | 无法判断左右侧障碍 | 控制层仅作为正前方避障；后续需要侧向避障时增加传感器。 |
| UART1 电机协议 | 仅确认 PA9/PA10，未提供波特率和协议帧 | 传感器层能提供数据，但不能直接定义电机命令格式 | 【已决策，需验证】传感器层只输出快照，电机协议另行设计。 |
| 旧功能回归范围 | 其他旧功能未说明 | 集成传感器后可能影响已有功能 | 最低回归：UART1 电机收发、I2C1 双传感器、主循环不卡顿、距离触发停车、IMU 姿态保护。 |

## 附录：后续低成本模型补代码时的执行顺序建议

1. 先建立 `jy61p.h/c`、`atk_ms53l1m.h/c`、`sensors.h/c` 文件骨架。
2. 先实现 I2C ready 检测和状态机，不急于实现所有滤波。
3. 实现 JY61P 连续读取 `0x34~0x40` 与物理量换算。
4. 实现 JY61P 车体系映射和符号宏。
5. 实现 JY61P 上电静止零偏校准。
6. 实现 VL53L1X 默认启动封装和距离读取。
7. 实现 VL53L1X 中值滤波、滑动平均、近距离优先。
8. 实现 `SENSOR_Snapshot_t` 快照接口。
9. 实现 I2C 短超时、错误计数、错误状态机。
10. 最后实现 I2C bus recovery 和长时间运行日志。
