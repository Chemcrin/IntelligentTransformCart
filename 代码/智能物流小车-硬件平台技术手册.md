## 1. 系统架构总览

### 1.1 架构分层

系统按「决策 — 控制 — 执行」三层划分，各层职责和物理连接如下：

```
┌──────────────────────────────────────────────────────────────────┐
│                    决策层：树莓派 4B                               │
│                                                                  │
│  · 视觉识别（杰瑞维通 DF100 摄像头）                                │
│  · 路径规划 / 任务调度                                             │
│  · 与底盘 STM32 双向通讯                                          │
│  · 运行树莓派4系统                                            │
└───────────────┬──────────────────────────────────────────────────┘
                │ UART（经板载 CH340-1 转 USB 虚拟串口）
                │ 物理层：3.3V TTL 点对点
                │ 协议：自定义二进制帧 [待定]
                │
┌───────────────▼──────────────────────────────────────────────────┐
│                  运动控制层：底盘 PCB                               │
│                                                                  │
│  MCU：STM32F103C8T6（Cortex-M3, 72MHz, 64KB Flash / 20KB RAM）    │
│                                                                  │
│  ┌──────────────────────────────────────────────────────────────┐│
│  │ 外设总线映射：                                                ││
│  │                                                              ││
│  │  UART1 ──── 与树莓派通讯（经板载 CH340-1 转 USB）             ││
│  │  UART3 ──── JY61P IMU 陀螺仪（UART, 115200bps, 3.3V TTL）   ││
│  │  I2C1  ──── MS53L1M 激光测距（IIC 直通模式, 地址 0x52）     ││
│  │  UART2 ──── RS485 总线 → 4× ZDT_X42 闭环步进驱动             ││
│  │  GPIO  ──── 急停按钮 / 状态 LED / 调试接口                    ││
│  │                                                              ││
│  └──────────────────────────────────────────────────────────────┘│
│                                                                  │
│  板载双路 CH340：                                                 │
│  · CH340-1：STM32 UART1 ↔ 树莓派 USB（主力通讯通道）              │
│  · CH340-2：悬空 / 保留（可接另一 UART 用于 PC 直连调试）         │
└───────────────┬──────────────────────────────────────────────────┘
                │ RS485 半双工差分总线
                │ 协议：Modbus-RTU（115200bps, 8N1）
                │ 从机地址：0x01 / 0x02 / 0x03 / 0x04
                │
┌───────────────▼──────────────────────────────────────────────────┐
│                执行层：4× ZDT_X42 闭环步进驱动                      │
│                                                                  │
│  · 控制模式：直通限速位置模式（推荐）/ 梯形曲线位置模式              │
│  · FOC 矢量闭环（力矩-速度-位置三环控制）                          │
│  · 板载 16384 线磁编码器                                          │
│  · 配合麦克纳姆轮组实现全向移动                                     │
│  · 供电：12V 锂电池直连（不经 DCDC）                               │
└──────────────────────────────────────────────────────────────────┘
```

### 1.2 数据流向

| 方向 | 路径 | 数据内容 |
|---|---|---|
| 上行（反馈） | 电机编码器 → RS485 → STM32 → UART → 树莓派 | 里程计、位置、速度反馈 |
| 上行（反馈） | JY61P → UART3 → STM32 → UART → 树莓派 | IMU 姿态数据（Roll/Pitch/Yaw） |
| 上行（反馈） | MS53L1M → I2C1 → STM32 → UART → 树莓派 | 障碍物距离 |
| 下行（控制） | 树莓派 → UART → STM32 → RS485 → 电机驱动 | 速度向量 / 目标位置指令 |

## 2. 主控层：树莓派 4B

### 2.1 角色定位

- 作为 **上位机**，运行 树莓派4 系统
- 运行视觉识别算法（杰瑞维通 DF100 摄像头输入）
- 全局路径规划与任务调度
- 向底盘 STM32 发送运动指令（速度向量 / 目标位置）
- 接收底盘 STM32 回传的传感器融合数据（里程计、IMU 姿态、障碍物警告）

### 2.2 对外接口

| 接口 | 连接对象 | 用途 | 备注 |
|---|---|---|---|
| USB | 底盘 PCB 板载 CH340-1 | 与底盘 STM32 通讯 | Linux 下识别为 `/dev/ttyUSB0` 或 `/dev/ttyACM0` |
| USB | DF100 摄像头 | 视觉识别输入 | 杰瑞维通 |
| USB | 键盘 / 鼠标 / 显示器 | 开发调试 | 部署后可移除 |

### 2.3 与底盘通讯协议（设计要点）

树莓派与底盘 STM32 之间通过 CH340 虚拟串口进行点对点通信，需设计自定义二进制帧协议，满足以下要求：

| 需求 | 说明 |
|---|---|
| **帧校验** | 携带 CRC（建议 CRC16 或 CRC32），确保数据完整性 |
| **指令类型** | 至少支持：运动指令（速度向量 / 目标位置）、状态查询、传感器数据回传 |
| **超时保护** | 树莓派死机或通讯断开时，底盘 STM32 必须在超时窗口内自主停车 |
| **心跳机制** | 建议双向心跳包，超时阈值建议 500ms-1s（视控制频率调整） |

### 2.4 CH340 在树莓派侧的技术说明

CH340 是 USB 转 TTL 串口芯片。树莓派通过 USB 接口连接板载 CH340-1，操作系统自动识别为串口设备。Linux 下使用 `pyserial` 等标准库即可进行串口通讯，无需额外驱动开发。

**物理接线路径**：

```
树莓派 USB --→ 底盘 PCB CH340-1 --→ STM32 UART1 (PA9/PA10, 3.3V TTL)
```

**关键设计要点**：

- 此链路为 **点对点 USB 转 TTL 串口**，不经 RS485 转换
- 电平为 **3.3V TTL**，两端无需额外电平转换
- 此通道仅用于两主控之间通讯，不挂载其他设备

## 3. 运动控制层：底盘 STM32F103C8T6

### 3.1 芯片基本信息

| 参数 | 值 |
|---|---|
| 内核 | ARM Cortex-M3 @ 72MHz |
| Flash | 64 KB |
| RAM | 20 KB |
| 板载资源 | 双路 CH340（USB 转 TTL） |

### 3.2 角色定位

- **强实时控制核心**：直接与电机驱动、IMU、激光传感器交互
- 执行 **麦克纳姆轮逆运动学解算**（速度向量 → 四轮 RPM）
- 运行 FreeRTOS 实时操作系统，承载各控制任务
- 实现 **故障降级**：树莓派通讯超时 → 自主安全停车
- 传感器数据采集与预处理（IMU 姿态融合、距离滤波）

### 3.3 关键工程约束

| 约束项 | 数值 | 说明 |
|---|---|---|
| RAM 总量 | 20 KB | **极其紧张**，需精确分配任务栈空间 |
| Flash 总量 | 64 KB | 充足，够第一阶段所有驱动 + 控制代码 |
| FreeRTOS 堆 | 建议 8-10 KB | 从 RAM 中分配 |
| 任务栈 | 用 `uxTaskGetStackHighWaterMark()` 实测后精确设定 | 不可凭经验估算 |
| 系统时钟 | 72MHz (HSE 8MHz × PLL) | 标准配置 |

### 3.4 引脚分配方案（草案）

| 外设 | STM32 引脚 | 功能 | 备注 |
|---|---|---|---|
| 树莓派通讯 | PA9 (TX), PA10 (RX) | USART1 | 经板载 CH340-1 转 USB |
| IMU (JY61P) | PB10 (TX), PB11 (RX) | USART3 | UART, 115200bps, 3.3V TTL 直连 |
| 激光 (MS53L1M) | PB6 (SCL), PB7 (SDA) | I2C1 | 直通 VL53L1X 传感器芯片 |
| RS485 总线 | PA2 (TX), PA3 (RX), PA1 (DE/RE) | USART2 | 经 SP3485 芯片转 RS485 |
| 急停按钮 | PB12 | GPIO 外部中断 | 上升沿 + 下降沿双触发 |
| 状态 LED | PC13 | GPIO 推挽输出 | 复用板载 LED（低电平点亮） |

### 3.5 板载双路 CH340 说明

小组自制 PCB 集成了双路 CH340，属 STM32 开发板常见设计。两路 CH340 的用途如下：

| CH340 通道 | 预期用途 | 连接方式 |
|---|---|---|
| CH340 | 树莓派 ↔ STM32 主力通讯 | TX/RX 接 STM32 UART1 (PA9/PA10) |





**PCB 电路设计要求**：

- 每个 CH340 需要独立的 **12MHz 外部晶振**（精确波特率所需），晶振应尽量靠近 CH340 芯片放置
- 电源去耦电容（**100nF + 10μF** 并联）靠近 VCC 引脚放置
- CH340 地线区域自成一体，最后单点连接至开发板主地线，防止数字噪声串 扰

## 4. IMU 传感器：维特智能 JY61P

### 4.1 基本信息

| 参数 | 值 |
|---|---|
| 型号 | JY61P |
| 传感器类型 | 三轴陀螺仪 + 三轴加速度计 + 三轴磁力计 |
| 姿态融合算法 | 9 轴卡尔曼滤波 |
| 姿态精度 | 动态 0.2°（Roll / Pitch），航向 0.5°（静态 6 轴模式） |
| 量程 | 加速度 ±16g，角速度 ±2000°/s |
| 供电电压 | 3.3V-5V |
| 工作电流 | ~8.4mA (@5V) |
| 数据输出速率 | 0.2Hz-200Hz 可选，默认 10Hz |

### 4.2 通信接口

**本阶段采用 UART 模式**。

| 参数 | 值 |
|---|---|
| 默认波特率 | 115200 bps |
| 帧格式 | 8 位数据位 / 无校验 / 1 停止位（8N1） |
| 输出内容 | 角度（Roll / Pitch / Yaw）、角速度、加速度（均可配置） |
| 建议输出速率 | 50Hz 或 100Hz（通过寄存器 0x01 RSW 和 0x03 RRATE 设置） |

### 4.3 接线方式

```
STM32F103                      JY61P
────────────────────────────────────────
PB10 (USART3 TX)  ──────────── RX  (引脚 4)
PB11 (USART3 RX)  ──────────── TX  (引脚 3)
3.3V              ──────────── VCC (引脚 2 / 11)
GND               ──────────── GND (引脚 5 / 8)
```

### 4.4 配置流程

JY61P 上电后自动进入数据输出模式（默认输出角度数据，10Hz）。建议在 STM32 初始化阶段通过串口发送配置指令：

1. **设置输出速率**：写 RSW 寄存器（0x02）选择输出内容，写 RRATE 寄存器（0x03）设置速率至 50Hz
2. **设置输出内容**：至少包含角度（数据码 0x53）、角速度（数据码 0x52）；如需惯导推算，额外开启加速度（数据码 0x51）
3. **航向角置零（校准）**：上电静止 2 秒后，发送 CALSW=0x04 指令将当前朝向设为 Yaw=0°

> **关于寄存器配置的说明**：JY61P 的寄存器修改遵循「解锁 → 写值 → 保存」三步流程，否则配置掉电丢失。但如果仅需读取实时数据（数据码 0x55 0x53 开头的数据包），传感器上电即按已保存的寄存器设置自动输出，**无需解锁流程**。解锁 → 写入 → 保存流程主要用于修改寄存器并使其掉电保存。建议首次使用时通过上位机软件检查并设置好寄存器值，之后每次上电传感器自动恢复已保存配置。

### 4.5 数据帧解析注意事项

JY61P 的 UART 数据帧有固定帧头标识，解析时需注意：

- 角度数据包以 `0x55 0x53` 开头，后跟 8 字节 Roll / Pitch / Yaw 数据（各 2 字节，有符号 16 位，单位为 0.01°）
- 角速度数据包以 `0x55 0x52` 开头
- 加速度数据包以 `0x55 0x51` 开头
- 每包末尾带 1 字节校验和（累加和，取低 8 位）

## 5. 激光测距模块：正点原子 ATK-MS53L1M

### 5.1 基本信息

| 参数 | 值 |
|---|---|
| 型号 | ATK-MS53L1M |
| 传感器芯片 | VL53L1X（ST 意法半导体） |
| 测量距离 | 40mm-4000mm（长距离模式） |
| 分辨率 | 1mm |
| 工作电压 | 3.3V-5V |
| 工作电流 | < 30mA |
| 视野角度（FOV） | 25° |
| 激光波长 | 940nm（Class 1 人眼安全） |

### 5.2 通信接口

**本阶段采用 I2C 模式通信**（直通 VL53L1X 传感器芯片）。

| 参数 | 值 |
|---|---|
| 工作模式 | IIC 模式（需通过上位机从 Normal 模式切换） |
| I2C 地址 | **0x52**（VL53L1X 默认 7bit 地址，左移后为 0xA4/0xA5） |
| 模块内上拉 | SDA / SCL 内部已接 10kΩ 上拉电阻 |
| 通信对象 | **直通 VL53L1X 传感器芯片**，不经过模块 MCU |

> **⚠️ 重要设计约束**：
>
> - IIC 模式下，模块 MCU 释放 VL53L1X 的 I2C 总线控制权，SDA / SCL 引脚直通传感器芯片 — 所有驱动代码须使用 **VL53L1X 原生寄存器**，不能使用手册中的 Modbus 指令
> - **校准功能在 IIC 模式下失效**，必须在 Modbus / UART 模式下用上位机完成校准后，再切回 IIC 模式使用
> - 测量模式（短距 / 中距 / 长距）也需在 IIC 模式下通过 VL53L1X 原生寄存器代码配置，或在切换前通过 Modbus 模式预设

### 5.3 接线方式

```
STM32F103                      MS53L1M
────────────────────────────────────────
PB6 (I2C1 SCL)  ───────────── 蓝色 (SCL)
PB7 (I2C1 SDA)  ───────────── 绿色 (SDA)
3.3V            ───────────── 红色 (VCC)
GND             ───────────── 黑色 (GND)
                              黄色 (TX) — 悬空（IIC 模式下不使用串口）
                              白色 (RX) — 悬空（IIC 模式下不使用串口）
```

### 5.4 从 Normal 模式切换到 IIC 模式

**所需工具**：USB-TTL 串口模块 + 正点原子上位机软件

**操作步骤**：

1. 用杜邦线连接 MS53L1M 模块到 USB-TTL 模块：

   ```
   USB-TTL 模块       MS53L1M
   ────────────────────────────
   TXD        ─────── 白色 (RX)
   RXD        ─────── 黄色 (TX)
   3.3V / 5V  ─────── 红色 (VCC)
   GND        ─────── 黑色 (GND)
   ```

2. 打开正点原子上位机软件，选择对应串口号，波特率 115200
3. 在「模块操作」中选择 **「TOF_4M」**，点击「搜索模块」，确认识别到模块
4. 点击「配置」→ 在「工作模式」中选择 **「IIC 模式」**→ 点击「设置」
5. 模式切换完成后，断开 USB-TTL，按 I2C 方式重新接线

> **警告**：模式切换后模块不再通过串口输出数据。如需重新用上位机配置，先通过串口切换回 Modbus 模式即可。极端情况下（如模块固件异常），说明书指出可将 SDA 接地后上电进入固件升级模式 — 但正常使用中极少需要此操作。

### 5.5 I2C 总线注意事项

- JY61P 改用 UART 后，I2C1 总线上 **仅有一个设备**（VL53L1X, 地址 0x52），不存在地址冲突
- MS53L1M 模块内部已有 10kΩ 上拉，STM32 侧可不额外加上拉电阻；若走线较长（>20cm）或速率较高（>400kHz），需用示波器确认上升沿是否干净
- 如使用 ST 官方 VL53L1X 驱动库（`VL53L1X_api.h`），需适配 HAL 的 `I2C_HandleTypeDef` 句柄

## 6. 麦克纳姆轮 + ZDT_X42 闭环步进驱动系统

### 6.1 机械参数

| 参数 | 值 | 来源 |
|---|---|---|
| 麦克纳姆轮直径 | 75 mm | 用户提供 |
| 单轮厚度 | 33 mm | 用户提供 |
| 轮距（左右麦轮外表面中心间距） | 320 mm | 用户测量 |
| 轴距（前后麦轮中心间距） | 220 mm | 用户测量 |

> **对逆运动学解算的影响**：轮距 \(L\) 和轴距 \(W\) 是麦克纳姆轮逆运动学矩阵的核心参数。输入速度向量 \((v_x, v_y, \omega)\) 通过以下关系映射为各轮转速：
>
> \[
> \begin{bmatrix} \omega_1 \\ \omega_2 \\ \omega_3 \\ \omega_4 \end{bmatrix}
> = \frac{1}{R} \begin{bmatrix}
> 1 & -1 & -(L+W) \\
> 1 &  1 &  (L+W) \\
> 1 &  1 & -(L+W) \\
> 1 & -1 &  (L+W)
> \end{bmatrix}
> \begin{bmatrix} v_x \\ v_y \\ \omega \end{bmatrix}
> \]
>
> 轮距和轴距的测量精度直接影响直线行走精度和自旋稳定度。建议机械装配完成后用辅助工具复核测量（如游标卡尺），误差控制在 ±2mm 以内。

### 6.2 驱动板基本信息

| 参数 | 值 |
|---|---|
| 型号 | ZDT_X42（闭环步进驱动） |
| 驱动数量 | 4 个（左前 / 右前 / 左后 / 右后） |
| 位置传感器 | 板载 16384 线磁编码器 |
| 控制模式 | FOC 矢量闭环（力矩 / 速度 / 位置三环控制） |
| 通讯方式 | **Modbus-RTU over RS485** |
| 协议兼容性 | X28 / X35 / X42 / X57 统一协议栈 |

**关于协议兼容性**：ZDT_X42 与 X57 使用完全统一的 Modbus-RTU 指令系统。可通过读取寄存器 `0x0010` 区分具体固件版本和硬件型号。用户提供的 X57 手册中所有 Modbus 指令（控制动作命令列表、读取参数命令列表、修改参数命令列表等）**均适用于 X42 驱动板**，无需额外查找 X42 专属文档。

### 6.3 RS485 总线通信参数

| 参数 | 值 |
|---|---|
| 物理层 | RS485 半双工差分 |
| 硬件要求 | 每个驱动板需插入 1 个 ZDT_RS485 小模块 |
| 协议 | Modbus-RTU |
| 波特率 | 115200 bps |
| 数据位 | 8 |
| 校验位 | None（8N1） |
| 停止位 | 1 |
| 从机地址 | 0x01 / 0x02 / 0x03 / 0x04（分别对应左前 / 右前 / 左后 / 右后） |

### 6.4 ZDT_RS485 模块与接线

每个 ZDT_X42 闭环驱动插针上需插入一个 **ZDT_RS485 专用转换小模块**，实现 TTL ↔ RS485 差分信号转换。接线拓扑：

```
STM32 侧                           ZDT_RS485 模块 ×4
┌──────────────────┐               ┌──────────────────────────┐
│ SP3485 (RS485 芯片)│   RS485 总线  │ 模块 1 │ 模块 2 │ 模块 3 │ 模块 4 │
│                   │              │        │        │        │        │
│ RO  → PA3 (RX)   │   A ─────────│── A ───│── A ───│── A ───│── A    │
│ DI  ← PA2 (TX)   │   B ─────────│── B ───│── B ───│── B ───│── B    │
│ DE/RE ← PA1      │              │        │        │        │        │
│ A   → 总线 A     │              └────────┴────────┴────────┴────────┘
│ B   → 总线 B     │
└──────────────────┘
```

**RS485 终端电阻要求**：

- 在 RS485 总线 **首尾两端** 各并联一个 **120Ω 终端电阻**，用于阻抗匹配，消除信号反射
- 短距离（小车内部走线 < 1 米）调试阶段可暂不加，但最终部署建议加装
- 若底盘 PCB 已设计终端电阻焊盘：**仅首尾两节点加装**，中间节点不装

### 6.5 推荐控制方案：直通限速位置模式

对于麦克纳姆轮全向移动场景，**直通限速位置模式** 是最适合的控制方式：

| 参数 | 说明 |
|---|---|
| 功能码 | `0xFB`（写多个寄存器，Modbus 标准功能码 `0x10`，寄存器地址 `0x00F0`） |
| 指令参数 | 方向（0=正转 / 1=反转）、速度（RPM）、位置角度（需 ×10 后输入） |
| 控制逻辑 | 速度值 = 逆运动学解算后的各轮目标转速 |

**多机同步运动**：如需四轮同时启动 / 停止，使用同步机制：

1. 先分别向四个驱动板发送运动指令（各驱动板缓存但不立即执行）
2. 发送广播指令 — 地址 `0x00`，功能码 `0xFF 0x66`（触发多机同步运动）
3. 所有已缓存命令的电机同时开始运动

### 6.6 细分设置建议

| 参数 | 建议值 | 说明 |
|---|---|---|
| 细分 | **16 细分**（3200 pulse/rev） | 默认值，分辨率 0.1125°/pulse，运动控制常用 |
| 闭环模式 | FOC 矢量闭环（CtrlMode = `CR_VFOC`） | 驱动器内部完成 PID 闭环，STM32 无需做电机 PID |

### 6.7 驱动器上电初始化流程

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 1. 上电   │──→│ 2. 等待   │──→│ 3. 校验   │──→│ 4. 置零   │
│ 12V 供电  │   │ 自检完成   │   │ 编码器    │   │ 位置角    │
│           │   │ (OLED 屏  │   │ 就绪状态   │   │ (写 0x000A│
│           │   │  正常显示) │   │ (读 0x3B) │   │  =0x0001) │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
```

1. 接通 12V 电源，等待驱动板自检（OLED 显示正常界面，约 1-2 秒）
2. 逐一读寄存器 `0x3B`（回零状态标志位），确认编码器校准就绪
3. 写寄存器 `0x000A = 0x0001`，将所有轮子当前位置清零（设为角度零点）
4. 初始化完成后，可发送速度 / 位置指令开始运动

### 8.2 DCDC 模块规格

| 参数 | 值 |
|---|---|
| 型号 | LM2596 带数显 |
| 输入电压 | 12V（航模锂电池） |
| 输出电压 | 可调（需分别设置为 5V 和 3.3V） |
| 类型 | 降压型（Buck）开关稳压器 |

### 8.3 电源完整性风险

> **⚠️ 电机启动时电源纹波未测量**

步进电机急启 / 急停时，12V 母线可能瞬间大幅跌落。如果 DCDC 前级未预留足够容量的电解电容作为缓冲储能，可能导致 MCU 掉电复位（Brown-out Reset）。

**建议措施**：

| 措施 | 参数建议 |
|---|---|
| DCDC 输入侧并联电解电容 | **470μF-1000μF / 25V** |
| DCDC 输出侧并联陶瓷电容 | **100nF + 10μF**（已经存在，但需确认焊接） |
| 验证方法 | 用示波器抓取电机全速启动时 3.3V 轨波形，确认纹波在 ±100mV 以内 |

### 8.4 电池电压监测

**当前无电池电压监测电路**。如需低电量保护（电池单体电压 < 3.5V 时触发），需设计以下电路：

```
12V 母线 → 分压电阻 (10kΩ + 3.3kΩ) → STM32 ADC 引脚 (如 PA0)
                                      │
                                      ├── 分压比：12V → 3.0V（ADC 安全范围内）
                                      │
                                      └── 建议并接 100nF 电容滤除噪声
```

此项为 **P2 优先级**，第一阶段可不实现，但建议在 PCB 上预留焊盘。
