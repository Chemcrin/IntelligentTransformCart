# 智能物流小车 工程整合指南

> 文档版本：V1.0  
> 最后更新：2026-05-26  
> 作用：基于 GPT 版传感器工程（ATK_JY_NewVer_GPT），整合桥接层与麦轮运动控制

---

## 1. 整体架构

```
主循环 (while(1))
    │
    ├── Board_App_Loop()
    │       ├── IMU_App_Loop()      ← 读 JY61P → 更新全局变量 g_imu_*
    │       └── Laser_App_Loop()    ← 读 VL53L1X → 更新全局变量 g_laser_*
    │
    ├── Sensor_GetSnapshot()        ← 桥接层：deg→rad, mm→m → SensorSnapshot_t
    │
    ├── Chassis_ControlTask()       ← 运动控制层（新建）
    │       ├── 读快照 → 航向保持 / 距离限速 / 安全检测
    │       └── Mecanum_InverseKinematics() → WheelRpm_t
    │
    └── Motor_Manager_Send()       ← Modbus：将轮速发到四轮电机
```

## 3. 在 CubeIDE 中的添加步骤

### 第 1 步：新建 STM32 工程

- 以 GPT 版工程为基础直接复制一份
- 或者新建 CubeMX 工程后，把 GPT 版的 `Core/Src` 和 `Core/Inc` 文件全部拷贝进去
- **注意保持** `main.h` 中 I2C1 的配置（PB6/PB7）

### 第 2 步：添加桥接层文件

将 `sensor_bridge.c` 复制到 `Core/Src/`，`sensor_bridge.h` 复制到 `Core/Inc/`

### 第 3 步：在 CubeIDE 中将新文件加入编译

```
Project → Properties → C/C++ Build → Settings → MCU GCC Compiler → Include paths
添加：Core/Inc

然后在 Project Explorer 中右键 Core/Src → Add Files... 添加新的 .c 文件
```

### 第 4 步：修改 main.c

```c
#include "board_app.h"
#include "sensor_bridge.h"
// #include "chassis_control.h"    // 后续添加

int main(void)
{
    HAL_Init();
    SystemClock_Config();
    MX_GPIO_Init();
    MX_I2C1_Init();
    MX_USART1_UART_Init();      /* Modbus 用 */
    // MX_USART2_UART_Init();    /* 树莓派通信，可选 */

    Board_App_Init();            /* 初始化 IMU + 激光 */

    while (1)
    {
        Board_App_Loop();        /* 更新传感器数据 */

        SensorSnapshot_t snap;
        Sensor_GetSnapshot(&snap);  /* 桥接层：获取单位转换后的快照 */

        /* 在这里插入 Chassis_ControlTask(&snap) */

        HAL_Delay(10);           /* 约 100 Hz 主循环 */
    }
}
```

## 4. 传感器数据流验证

加完桥接层后，先在 main loop 里验证数据是否畅通：

```c
// main.c —— 验证代码
void Debug_PrintSnapshot(void)
{
    SensorSnapshot_t snap;
    Sensor_GetSnapshot(&snap);

    printf("yaw=%.1f deg (%.3f rad), ", snap.yaw_rad * 57.3f, snap.yaw_rad);
    printf("gyro_z=%.1f dps (%.3f rad/s), ", snap.gyro_z_rads * 57.3f, snap.gyro_z_rads);
    printf("dist=%.0f mm (%.3f m), ", snap.distance_m * 1000.0f, snap.distance_m);
    printf("warning=%d, imu_ok=%d, laser_ok=%d\r\n",
           snap.laser_warning, snap.imu_valid, snap.laser_valid);
}
```

预期输出（转车时）：
```
yaw=45.2 deg (0.789 rad), gyro_z=12.5 dps (0.218 rad/s), dist=320 mm (0.320 m), warning=2, imu_ok=1, laser_ok=1
```

## 6. 注意事项

| 注意点 | 说明 |
|---|---|
| **I2C 总线** | JY61P + VL53L1X 共用 I2C1（PB6/PB7），不要同时占用 |
| **IMU 安装方向** | GPT 版默认 +Y 向前，+X 向右。如果实际安装方向不同，改 `jy61p.h` 中的 `JY61P_SIGN_CAR_*` 宏 |
| **Modbus 优先级** | 电机控制通过 USART1 走 RS485，和传感器 I2C 两条独立链路，不冲突 |
| **主循环频率** | 建议 50~100 Hz（10~20 ms），太快则 I2C 频繁读取可能增加噪声 |
| **快照一致性** | `Board_App_Loop()` 先更新传感器，`Sensor_GetSnapshot()` 紧接着读取，保证快照内所有数据来自同一时刻 |
| **调试串口** | 建议用 USART2（PA2/PA3）接 CH340N 打印调试信息，不干扰 USART1 的 Modbus 通信 |
