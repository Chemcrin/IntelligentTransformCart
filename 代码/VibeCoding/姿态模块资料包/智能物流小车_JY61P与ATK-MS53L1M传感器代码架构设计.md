## 3. 总体设计目标

JY61P 负责读取智能物流搬运小车的姿态与惯性数据，包括偏航角、俯仰角、横滚角、三轴角速度、三轴加速度和温度。ATK-MS53L1M 负责车头前方单点激光测距，输出前向障碍距离、测量状态和数据更新时间。

两个传感器的数据需要统一缓存，供后续电机控制、避障、姿态修正、路径控制、安全停车和上位机遥测使用。设计重点如下：

| 目标 | 说明 |
|---|---|
| 实时性 | JY61P 10-20 ms 更新；激光 20 ms 轮询 ready，约 150 ms 刷新一次有效距离。 |
| 非阻塞 | 主循环不能被长时间 I2C 等待卡住；I2C 超时要短，失败后快速返回。 |
| 可诊断 | 每个模块有状态机、错误计数、最近错误码、更新时间戳。 |
| 数据稳定 | 姿态做零偏校准和轻量滤波；距离做中值/滑动平均/异常值剔除。 |
| 上层友好 | 电机控制层读取快照，不直接访问 I2C 或底层驱动。 |

## 7. JY61P 架构设计

### 7.1 通信方式

JY61P 按 IIC 寄存器访问方式设计。默认 7-bit 地址 `0x50`，HAL 调用时使用 `0x50 << 1`。

注意：JY61P 也支持 UART 输出 `0x55` 帧，但本文不按 UART 主方案设计。若实物 TX/RX 已接到 STM32 串口，则 JY61P 模块需要改成 UART 解析状态机，标记为 `待确认`。

### 7.2 初始化流程

| 步骤 | 操作 |
|---|---|
| 1 | `jy61p_init(&jy61p_handle, &hi2c1)` 保存 I2C 句柄，清空状态、错误计数、滤波缓存。 |
| 2 | 等待模块启动，JY61P 规格书启动时间约 1000 ms。实际代码可在系统上电后整体延时，或状态机等待。 |
| 3 | 通过 I2C 读取版本寄存器 `0x2E` 或读取 `0x34` 起始数据，验证设备有响应。 |
| 4 | 读取一次加速度、角速度、姿态角、温度，检查数值范围。 |
| 5 | 状态进入 `JY61P_STATE_RUNNING`。若上电要求静止校准，则进入 `JY61P_STATE_CALIBRATING`。 |

初始化阶段默认不要写 JY61P 内部配置寄存器，避免误改模块参数。只有在明确需要改地址、校准或保存参数时，才执行解锁和保存流程。

### 7.3 数据寄存器与转换

| 数据 | 寄存器 | 类型 | 单位 | 转换 |
|---|---:|---|---|---|
| 加速度 X | `0x34` | `int16_t` | g | `raw / 32768 * 16` |
| 加速度 Y | `0x35` | `int16_t` | g | `raw / 32768 * 16` |
| 加速度 Z | `0x36` | `int16_t` | g | `raw / 32768 * 16` |
| 角速度 X | `0x37` | `int16_t` | deg/s | `raw / 32768 * 2000` |
| 角速度 Y | `0x38` | `int16_t` | deg/s | `raw / 32768 * 2000` |
| 角速度 Z | `0x39` | `int16_t` | deg/s | `raw / 32768 * 2000` |
| 磁场 X/Y/Z | `0x3A-0x3C` | `int16_t` | LSB | 当前保留原始值，是否参与航向修正 `待确认`。 |
| Roll | `0x3D` | `int16_t` | deg | `raw / 32768 * 180` |
| Pitch | `0x3E` | `int16_t` | deg | `raw / 32768 * 180` |
| Yaw | `0x3F` | `int16_t` | deg | `raw / 32768 * 180` |
| 温度 | `0x40` | `int16_t` | ℃ | `raw / 100` |

### 7.4 数据接收方式建议

| 方式 | 建议 | 原因 |
|---|---|---|
| 轮询 | 初版推荐 | 每 10-20 ms 做一次短 I2C 读取，逻辑简单，便于调试。 |
| 中断 | 初版不推荐 | HAL I2C 中断状态机复杂，和同总线激光模块容易产生资源争用。 |
| DMA | 后续优化 | 只有在主循环阻塞明显、数据量增大时再引入。 |

### 7.5 数据有效性判断

JY61P IIC 直接读取寄存器时没有 UART 帧校验，因此必须从以下方面判断有效性：

| 检查项 | 规则 |
|---|---|
| HAL 返回值 | I2C 读写必须返回 `HAL_OK`，否则本次数据无效。 |
| 数值范围 | Roll/Yaw 应在约 `-180~180 deg`，Pitch 应在约 `-90~90 deg` 或按模块输出范围判断。 |
| 加速度模长 | 静止时 `sqrt(ax^2+ay^2+az^2)` 应接近 1g；运动时允许偏离。 |
| 时间戳 | 超过 `JY61P_DATA_TIMEOUT_MS` 未更新则数据过期。 |
| 连续性 | 姿态角单次跳变过大时标记异常，除非角速度数据支持该变化。 |

### 7.6 滤波策略

| 数据 | 建议滤波 |
|---|---|
| 加速度 | 一阶低通，默认 `alpha=0.2-0.4`，用于降低震动噪声。 |
| 角速度 | 一阶低通，默认 `alpha=0.2-0.4`，校准时使用滤波或均值。 |
| Roll/Pitch | 轻量滑动平均 3-5 点，避免过大延迟。 |
| Yaw | 不建议强滤波；控制直线时使用相对 yaw，避免响应变慢。 |

滤波只能在新数据有效时更新。数据无效或超时时，保持最后稳定值，并将 `valid` 置 0 或 `stale` 置 1。

### 7.7 上电静止校准流程

| 步骤 | 说明 |
|---|---|
| 进入条件 | 小车上电后未允许运动，电机目标速度为 0，编码器速度为 0 或电机未使能。 |
| 静止判定 | 连续 500-1000 ms 内三轴角速度绝对值小于阈值，且加速度模长接近 1g。 |
| 推荐阈值 | `abs(gx/gy/gz) < 1.0 deg/s`；`abs(acc_norm_g - 1.0) < 0.15g`，需实测调整。 |
| 采样次数 | 100-200 组，周期 10 ms。 |
| 平均计算 | 计算 `gx/gy/gz` 均值作为软件零偏，记录当前 yaw 为 `yaw_reference_deg`。 |
| 保存策略 | 默认只保存到 RAM，不写 JY61P 内部寄存器；避免频繁写模块 Flash 或误改参数。 |
| 完成条件 | 采样数量达标且数据稳定，状态转 `RUNNING`。 |
| 失败条件 | 采样期间检测到运动、I2C 错误过多或超时，状态转 `DATA_INVALID` 或 `COMM_ERROR`。 |

### 7.8 运行中累计误差抑制策略

| 场景 | 策略 |
|---|---|
| 小车静止 | 若电机目标速度为 0、编码器速度为 0、角速度持续低于阈值，则缓慢更新软件零偏，更新增益建议 `0.001-0.01`。 |
| 直线行驶 | 不直接将 yaw 清零；记录直线起点 yaw，使用当前 yaw 与起点 yaw 的相对误差做 PID 纠偏。 |
| 转弯期间 | 禁止更新零偏，避免把真实转向角速度当成漂移。 |
| 长时间运行 | 每个任务段开始时调用 `jy61p_set_yaw_reference`，使用相对 yaw 控制。 |
| 外部融合 | 若后续有视觉、码盘里程计或磁力计标定，可由上层融合修正 yaw，传感器层不做复杂融合。 |
| 手动置零 | 可通过 JY61P `CALSW=0x04` 航向角置零，但只允许在小车静止且用户明确触发时执行。 |

### 7.9 与运动控制模块接口

JY61P 模块只提供姿态、角速度、加速度、温度、有效标志、时间戳和校准状态，不直接输出电机 PWM 或速度命令。直线纠偏、坡度判断、异常停车应由运动控制层或安全状态机完成。

## 8. ATK-MS53L1M 架构设计

### 8.1 型号与通信方式

本文按 ATK-MS53L1M 内部 VL53L1X IIC 方案设计。

| 项目 | 值 |
|---|---|
| I2C 7-bit 地址 | `0x29` |
| HAL 8-bit 地址 | `0x52` |
| 模块 ID 寄存器 | `0x010F` |
| 期望 ID | `0xEACC` |
| 距离单位 | mm |
| 推荐模式 | Long |

### 8.2 初始化流程

| 步骤  | 操作                                                 |
| --- | -------------------------------------------------- |
| 1   | `ms53l1m_init(&ms53l1m_handle, &hi2c1)` 保存 I2C 句柄。 |
| 2   | 使用 `HAL_I2C_IsDeviceReady` 检测 `0x52` 是否响应。         |
| 3   | 读取 `0x010F`，检查是否为 `0xEACC`。                        |
| 4   | 调用 VL53L1X 初始化流程：等待启动、DataInit、StaticInit。         |
| 5   | 设置距离模式为 Long。                                      |
| 6   | 设置 Timing Budget 为 140000 us。                      |
| 7   | 设置 Inter Measurement Period 为 150 ms。              |
| 8   | 启动连续测量。                                            |
| 9   | 状态转 `MS53L1M_STATE_RUNNING`。                       |

### 8.3 单次测量与连续测量

| 模式 | 建议 |
|---|---|
| 单次测量 | 不推荐作为初版主方案；每次启动/等待/读取会增加阻塞和延迟。 |
| 连续测量 | 推荐；初始化后持续测量，主循环只轮询 DataReady，ready 后读取结果并清中断启动下一次。 |

### 8.4 测距数据有效性判断

| 检查项 | 规则 |
|---|---|
| HAL/VL53L1X 返回值 | 任一底层错误都不更新稳定距离。 |
| DataReady | 只有 ready 为真才读取；not ready 不是错误。 |
| RangeStatus | 只有有效状态才更新 `filtered_distance_mm`。 |
| 距离范围 | 小于 40 mm 或大于 3900 mm 时标记无效；无镜片 4000 mm 上限 `待确认`。 |
| 时间戳 | 超过 `MS53LM_DATA_TIMEOUT_MS` 未获得有效数据，状态转 `DATA_TIMEOUT`。 |
| 跳变 | 与上一稳定值差值超过 `MS53LM_JUMP_REJECT_MM` 时先标记 suspect，不立即覆盖稳定值。 |

### 8.5 滤波与稳定输出

| 处理 | 设计 |
|---|---|
| 原始值 | 每次有效测量保存 `raw_distance_mm`。 |
| 中值滤波 | 保存最近 5 个有效样本，取中值抵抗单点毛刺。 |
| 滑动平均 | 对中值后的结果做 3-5 点平均，输出 `filtered_distance_mm`。 |
| 异常剔除 | 若新值与上次稳定值跳变超过 300-500 mm，且只出现 1 次，先标记 suspect。 |
| 靠近障碍 | 进入危险距离可快速响应；解除危险需连续 2-3 次有效距离恢复，避免抖动。 |
| 单传感器限制 | 单点激光只能检测正前方局部区域，不能替代视觉或多方向避障。 |

### 8.6 向上层提供的数据

| 字段 | 说明 |
|---|---|
| `raw_distance_mm` | 最新一次原始有效距离。 |
| `filtered_distance_mm` | 滤波后稳定距离，供控制和避障使用。 |
| `range_status` | VL53L1X/模块测量状态。 |
| `valid` | 当前稳定值是否有效。 |
| `too_close` | 是否小于停车阈值。 |
| `last_update_ms` | 最近一次有效距离更新时间。 |
| `sample_count` | 有效样本计数。 |
| `error_count` | I2C/VL53L1X 错误计数。 |

## 9. 持续测量与实时更新机制

### 9.1 初始化阶段

1. `HAL_Init()`。
2. 系统时钟初始化。
3. `MX_GPIO_Init()`。
4. `MX_I2C1_Init()`，PB6/PB7。
5. `sensor_init_all(&hi2c1)`。
6. JY61P 初始化，读取首帧。
7. MS53L1M 初始化，启动连续测量。
8. 若上电要求校准，执行 JY61P 静止校准，校准期间禁止电机运动。

### 9.2 主循环阶段

主循环每轮调用统一调度函数。调度函数根据 `HAL_GetTick()` 判断是否到达各模块更新时间：

| 任务               |      周期建议 | 行为                           |
| ---------------- | --------: | ---------------------------- |
| JY61P 更新         |  10-20 ms | 读取寄存器、转换物理量、滤波、更新时间戳。        |
| MS53L1M ready 轮询 |     20 ms | 查询 DataReady；ready 后读取距离并滤波。 |
| 传感器健康检查          |    1-5 ms | 检查超时、错误计数、状态机转换。             |
| 状态上报             | 20-100 ms | 将快照给树莓派或调试输出。                |

### 9.3 数据缓存方式

底层模块维护自己的最新数据和状态，统一层提供快照。上层不得直接访问底层静态变量，也不得直接调用 I2C 读写函数。

| 缓存 | 更新者 | 读取者 |
|---|---|---|
| `JY61P_Attitude_t` | `jy61p_update` | `sensor_get_snapshot`、运动控制层。 |
| `JY61P_Motion_t` | `jy61p_update` | 安全状态机、运动控制层。 |
| `MS53LM_Distance_t` | `ms53lm_update` | 避障、安全状态机。 |
| `SENSOR_SystemState_t` | `sensor_update_all` | 电机控制、遥测、调试。 |

### 9.4 避免阻塞主循环

| 风险 | 处理 |
|---|---|
| I2C 读写卡住 | HAL 超时设置为 5-10 ms，不允许无限等待。 |
| 激光等待测量完成 | 不使用阻塞等待，改为 DataReady 轮询。 |
| 校准耗时 | 校准做成状态机，多次采样分散在主循环中，不一次性阻塞 2 秒。 |
| I2C 总线卡死 | 记录错误，必要时执行 bus recovery：临时 GPIO 输出 SCL 产生 9 个时钟，再重新初始化 I2C。 |

## 10. 与电机控制模块的配合方式

| 数据 | 用法 | 建议归属 |
|---|---|---|
| `yaw_deg` | 直线行驶纠偏，计算当前 yaw 与起始 yaw 的偏差。 | 运动控制层做 PID；传感器层只提供角度。 |
| `pitch_deg` | 判断坡度、上坡/下坡、异常翘头。 | 传感器层提供滤波值，上层判断阈值。 |
| `roll_deg` | 判断侧倾、碰撞、倾倒风险。 | 安全状态机处理。 |
| `ax/ay/az_g` | 判断启动、急停、震动、碰撞。 | 上层事件判断。 |
| `gx/gy/gz_dps` | 判断旋转速度、转向是否异常。 | 运动控制层使用。 |
| `front_distance_mm` | 避障、减速、停车。 | 传感器层提供 raw/filtered/status，上层生成限速或停车命令。 |

推荐控制逻辑：

| 场景 | 控制策略 |
|---|---|
| 直线行驶 | 记录起始 yaw，实时计算 yaw error，电机差速/麦轮控制用 PID 修正。 |
| 靠近障碍 | 距离小于预警阈值时限速，距离小于硬停车阈值时停车。 |
| 坡度变化 | Pitch 超过阈值时限速，并上报地形变化。 |
| 姿态异常 | Roll/Pitch 或角速度超过安全阈值时停车或降级。 |
| 传感器超时 | IMU 超时禁用姿态闭环；激光超时禁用自动前进或降速。 |

## 11. 状态机与错误处理

### 11.1 通用状态机

| 状态 | 进入条件 | 退出条件 | 上层处理 |
|---|---|---|---|
| `UNINIT` | 上电默认 | 调用 init | 不使用数据。 |
| `INITING` | 正在初始化 | 成功转 `RUNNING`，失败转 `COMM_ERROR` | 禁止运动。 |
| `RUNNING` | 初始化成功且数据有效 | 超时、错误或校准请求 | 正常使用数据。 |
| `CALIBRATING` | 用户或上电触发校准 | 校准完成或失败 | 电机保持停止。 |
| `DATA_TIMEOUT` | 超过超时阈值无新数据 | 恢复连续有效样本 | 可短时保留旧值，但必须标记过期。 |
| `COMM_ERROR` | HAL I2C 错误或设备无响应 | recover 成功 | 不使用新数据，保留最后有效值用于诊断。 |
| `DATA_INVALID` | 数值越界、状态无效、跳变异常 | 连续有效样本恢复 | 不直接覆盖稳定值。 |
| `NEED_REINIT` | 连续错误超过阈值 | 重新初始化成功 | 上层降级，必要时停车。 |

### 11.2 是否允许继续使用旧数据

| 状态 | 是否允许使用旧数据 | 说明 |
|---|---|---|
| `RUNNING` | 是 | 正常使用最新数据。 |
| `DATA_INVALID` | 有条件 | 可保留上次稳定值，但 `valid=0` 或 `stale=1`。 |
| `DATA_TIMEOUT` | 短时有条件 | IMU 可短时用于显示，不能用于闭环；激光超时不应用于放行前进。 |
| `COMM_ERROR` | 否 | 仅可用于诊断显示。 |
| `CALIBRATING` | 否 | 校准期间控制层应停止或保持锁定。 |
| `NEED_REINIT` | 否 | 必须恢复后再使用。 |

## 13. 推荐数据结构设计

### 13.1 JY61P 状态枚举

| 枚举名 | 含义 |

| `JY61P_STATE_UNINIT` | 未初始化。 |
| `JY61P_STATE_INITING` | 初始化中。 |
| `JY61P_STATE_RUNNING` | 正常运行。 |
| `JY61P_STATE_CALIBRATING` | 校准中。 |

## 15. 文件组织建议

推荐目录结构：

```text
Core/
  Inc/
    jy61p.h
    atk_ms53lm.h
    sensors.h
  Src/
    jy61p.c
    atk_ms53lm.c
    sensors.c
Drivers/
  VL53L1X/
    core/
    platform/
```

若继承现有 ATK 框架文件名 `atk_ms53l1m.h/.c`，可以保持该名称，但必须全工程一致，不要同时存在 `atk_ms53lm` 和 `atk_ms53l1m` 两套 API。

### 15.1 CubeIDE 生成代码注意事项

| 项目 | 要求 |
|---|---|
| `main.c` | 用户调用代码必须放在 `USER CODE BEGIN` / `USER CODE END` 区域。 |
| `i2c.c` | CubeMX 生成 I2C1 初始化，PB6/PB7，不应手动写在用户驱动里。 |
| 新增驱动文件 | `jy61p.c/h`、`atk_ms53lm.c/h`、`sensors.c/h` 不应被 CubeMX 覆盖。 |
| Include Path | 若使用 VL53L1X API，需要把 `Drivers/VL53L1X/core/inc` 和 `Drivers/VL53L1X/platform/inc` 加入 include path。 |

### 15.2 main.c 调用流程

```text
HAL_Init
SystemClock_Config
MX_GPIO_Init
MX_I2C1_Init
sensor_init_all(&hi2c1)

while (1)
{
    sensor_update_all(HAL_GetTick())
    motor_control_update()
    telemetry_update()
}
```

注意：以上是伪流程，不是 `.c` 实现代码。

## 18. 未验证风险

| 风险                            | 影响                        | 建议处理方式                              |
| ----------------------------- | ------------------------- | ----------------------------------- |
| PB6/PB7 上拉电压不确定               | 若上拉到 5V，可能损坏或影响 STM32 I2C | 必须万用表/示波器确认；必要时改 3.3V 上拉或加电平转换。     |
| HAL 库版本未确认                    | API 和工程配置可能有差异            | 后续实现前固定 STM32CubeF1 版本，并记录在 README。 |
| 是否使用 RTOS/DMA 未确认             | 调度和数据保护方式不同               | 初版按裸机轮询；后续再迁移任务或 DMA。               |
| 电机控制接口未定义                     | 传感器到控制的字段可能对接不顺           | 传感器层只提供快照，不耦合电机实现。                  |
| IMU 坐标系与车体坐标系未标定              | 姿态方向可能反，导致纠偏错误            | 安装后做方向标定表，必要时在 `sensors` 层做轴映射。     |
| 实车震动影响姿态稳定性                   | 角速度和加速度噪声增加，yaw 漂移加剧      | 实测调整 JY61P 滤波、静止阈值、校准策略。            |
| 单点激光视场有限                      | 只能检测正前方小区域，可能漏检侧向障碍       | 只作为前向辅助；结合视觉或多方向传感器。                |
| I2C 共总线卡死                     | 一个设备异常可能影响另一个设备           | 加短超时、错误计数、bus recovery 和重新初始化策略。    |

## 19. 后续代码实现注意事项

1. 后续模型必须先确认最终工程路径，再创建或修改 `Core/Inc`、`Core/Src` 文件。
2. 不要在驱动中重复初始化 I2C1，只接收 `hi2c1` 指针。
3. 不要在传感器层直接控制电机。
4. 所有对上层输出的数据必须带 `valid` 和 `last_update_ms`。
5. 所有未确认参数必须保留宏定义，方便实测后修改。
6. 对 JY61P 内部寄存器写操作必须经过解锁、写目标、保存三个步骤，且默认不要自动写内部参数。
7. 对 MS53LM 距离数据，只有状态有效且范围合理时才更新稳定距离。
8. 初版优先保证稳定可调试，再考虑 I2C DMA 或 RTOS。
