# 传感器数据变量与 CubeIDE 调试说明

本文档对应代码目录：

```text
D:\AI_agent\Code\stm32cubeide_atk_ms53l1m_i2c_framework
```

当前工程集成两个传感器：

- ATK-MS53L1M 激光测距模块：使用 `I2C1`，引脚 `PB8/PB9`
- JY61P 陀螺仪/姿态传感器：使用 `USART3`，引脚 `PB10/PB11`

最终给 `main.c` 调用的是统一入口：

```c
#include "board_app.h"
```

## 1. 文件分层

| 层级 | 文件 | 作用 |
| --- | --- | --- |
| 板级统一入口 | `Core/Inc/board_app.h`, `Core/Src/board_app.c` | 统一初始化、统一循环调用、转发 JY61P UART 接收回调 |
| 激光应用层 | `Core/Inc/laser_app.h`, `Core/Src/laser_app.c` | 保存激光距离、状态、采样次数 |
| 激光底层驱动 | `Core/Inc/atk_ms53l1m.h`, `Core/Src/atk_ms53l1m.c` | 调用 ST VL53L1X API，从 ATK-MS53L1M 读取距离 |
| 陀螺仪应用层 | `Core/Inc/imu_app.h`, `Core/Src/imu_app.c` | 保存姿态角、加速度、角速度、数据包计数 |
| 陀螺仪底层驱动 | `Core/Inc/jy61p.h`, `Core/Src/jy61p.c` | 解析 JY61P UART 数据帧，校验并换算单位 |

## 2. 板级统一变量

这些变量用于判断整个应用框架是否在运行。

| 变量名 | 类型 | 定义位置 | 声明位置 | 含义 |
| --- | --- | --- | --- | --- |
| `g_board_app_flags` | `volatile uint8_t` | `Core/Src/board_app.c` | `Core/Inc/board_app.h` | 板级应用标志位。当前使用 `BOARD_APP_FLAG_INIT_DONE = 0x01` 表示已执行 `Board_App_Init()` |
| `g_board_app_loop_count` | `volatile uint32_t` | `Core/Src/board_app.c` | `Core/Inc/board_app.h` | `Board_App_Loop()` 被调用的次数。持续增加说明主循环在运行 |
| `g_board_app_last_tick_ms` | `volatile uint32_t` | `Core/Src/board_app.c` | `Core/Inc/board_app.h` | 最近一次执行 `Board_App_Loop()` 的 `HAL_GetTick()` 时间戳，单位 ms |

调试时优先观察：

```text
g_board_app_loop_count
```

如果这个变量不增加，说明 `while (1)` 中没有持续调用 `Board_App_Loop()`，后续传感器变量也不会更新。

## 3. 激光传感器公开变量

这些变量是后续小车控制逻辑应直接读取的激光数据。

| 变量名 | 类型 | 定义位置 | 声明位置 | 含义 |
| --- | --- | --- | --- | --- |
| `g_laser_status` | `volatile ATK_MS53L1M_StatusTypeDef` | `Core/Src/laser_app.c` | `Core/Inc/laser_app.h` | 激光最近一次初始化或读取状态 |
| `g_laser_distance_mm` | `volatile uint16_t` | `Core/Src/laser_app.c` | `Core/Inc/laser_app.h` | 最新激光测距值，单位 mm |
| `g_laser_module_id` | `volatile uint16_t` | `Core/Src/laser_app.c` | `Core/Inc/laser_app.h` | 模块 ID，正常应为 `0xEACC` |
| `g_laser_sample_count` | `volatile uint32_t` | `Core/Src/laser_app.c` | `Core/Inc/laser_app.h` | 激光成功读取次数。持续增加说明激光读取正常 |
| `g_laser_last_tick_ms` | `volatile uint32_t` | `Core/Src/laser_app.c` | `Core/Inc/laser_app.h` | 最近一次成功读取激光距离的时间戳，单位 ms |

`g_laser_status` 的枚举定义在：

```text
Core/Inc/atk_ms53l1m.h
```

常用状态：

| 状态 | 数值 | 含义 |
| --- | --- | --- |
| `ATK_MS53L1M_OK` | `0` | 正常 |
| `ATK_MS53L1M_ERROR_HAL` | `1` | HAL 参数或调用错误 |
| `ATK_MS53L1M_ERROR_NOT_FOUND` | `2` | 未检测到模块或模块 ID 不匹配 |
| `ATK_MS53L1M_ERROR_VL53L1` | `3` | VL53L1X API 返回错误 |
| `ATK_MS53L1M_ERROR_NOT_READY` | `4` | 本次测距数据尚未准备好 |

使用示例：

```c
if (g_laser_sample_count > 0U)
{
    uint16_t distance_mm = g_laser_distance_mm;
}
```

## 5. JY61P 底层静态缓存变量

这些变量定义在：

```text
Core/Src/jy61p.c
```

它们带有 `static`，只能在 `jy61p.c` 内部访问，不能直接在其他 `.c` 文件中使用。

| 变量名 | 类型 | 含义 |
| --- | --- | --- |
| `g_jy61p_huart` | `static UART_HandleTypeDef *` | 当前绑定的 UART 句柄，来自 `JY61P_Init(&huart3)` |
| `g_uart_rx_byte` | `static uint8_t` | HAL 单字节中断接收缓冲 |
| `g_rx_buf[10]` | `static uint8_t[]` | JY61P 数据帧缓冲，不包含帧头 `0x55` |
| `g_rx_idx` | `static uint8_t` | 当前接收索引 |
| `g_rx_locked` | `static uint8_t` | 是否已经检测到帧头 |
| `g_data` | `static JY61P_DataTypeDef` | 底层解析后的最新一组 IMU 数据 |
| `g_ready_flags` | `static volatile uint8_t` | 底层新数据标志 |
| `g_packet_count` | `static volatile uint32_t` | 底层成功解析包数 |
| `g_checksum_error_count` | `static volatile uint32_t` | 底层校验错误数 |

对外读取这些底层数据时不要直接访问 `static` 变量，而是通过：

```c
JY61P_GetData(&imu);
JY61P_GetPacketCount();
JY61P_GetChecksumErrorCount();
```

`imu_app.c` 已经调用这些接口，并把结果复制到 `g_imu_*` 公开变量。

## 6. main.c 如何调用

### 6.1 头文件

在 `Core/Src/main.c` 的 USER CODE include 区加入：

```c
#include "board_app.h"
```

### 6.2 初始化顺序

在 CubeMX 生成的外设初始化之后调用 `Board_App_Init()`：

```c
MX_GPIO_Init();
MX_I2C1_Init();
MX_USART3_UART_Init();

Board_App_Init();
```

要求：

- `MX_I2C1_Init()` 必须在 `Board_App_Init()` 之前调用，因为激光驱动需要 `hi2c1`
- `MX_USART3_UART_Init()` 必须在 `Board_App_Init()` 之前调用，因为 JY61P 驱动需要 `huart3`
- `USART3 global interrupt` 必须在 CubeMX 的 NVIC 中启用，否则 JY61P 不会接收到数据

### 6.3 UART 接收回调

在 `main.c` 中添加或合并此回调：

```c
void HAL_UART_RxCpltCallback(UART_HandleTypeDef *huart)
{
    Board_App_UartRxCpltCallback(huart);
}
```

如果工程里已经有 `HAL_UART_RxCpltCallback()`，不要重复定义第二个同名函数。应该把下面这一行合并到已有函数内部：

```c
Board_App_UartRxCpltCallback(huart);
```

### 6.4 主循环

在 `while (1)` 中持续调用：

```c
while (1)
{
    Board_App_Loop();

    // 在这里写小车控制逻辑
}
```

`Board_App_Loop()` 内部会调用：

```c
IMU_App_Loop();
Laser_App_Loop();
```

其中：

- `IMU_App_Loop()` 将 JY61P 底层解析结果复制到 `g_imu_*`
- `Laser_App_Loop()` 周期性读取激光距离，并更新 `g_laser_*`

### 6.5 控制代码读取示例

```c
Board_App_Loop();

if ((g_laser_sample_count > 0U) && (g_imu_packet_count > 0U))
{
    uint16_t distance_mm = g_laser_distance_mm;
    float yaw_deg = g_imu_yaw_deg;
    float gz_dps = g_imu_gz_dps;

    // 根据 distance_mm, yaw_deg, gz_dps 控制电机
}
```

## 7. CubeIDE 调试读取数据步骤

### 7.1 生成和烧录前检查

在 CubeMX / CubeIDE 中确认：

1. `I2C1` 已启用，并 remap 到 `PB8/PB9`
2. `USART3` 已启用，`PB10 = USART3_TX`, `PB11 = USART3_RX`
3. `USART3` 参数为 `115200-8-N-1`
4. `USART3 global interrupt` 已启用
5. `PB6/PB7` 不要配置为 `I2C1`
6. 工程 include path 包含：

```text
Core/Inc
Drivers/VL53L1X/core/inc
Drivers/VL53L1X/platform/inc
```

### 7.2 进入 Debug

1. 用 ST-LINK 连接 STM32
2. 在 CubeIDE 点击 `Debug`
3. 进入 Debug 透视图后，程序通常会停在 `main()`
4. 点击 `Resume`，让程序开始运行

### 7.3 打开 Live Expressions

如果界面没有显示 Live Expressions：

```text
Window -> Show View -> Expressions
```

或：

```text
Window -> Show View -> Live Expressions
```

不同版本 CubeIDE 菜单名称略有差异。推荐使用 `Live Expressions`，它会在程序运行时周期刷新变量。

### 7.4 添加变量

在 Expressions 或 Live Expressions 中逐个添加：

```text
g_board_app_loop_count
g_laser_status
g_laser_module_id
g_laser_distance_mm
g_laser_sample_count
g_imu_roll_deg
g_imu_pitch_deg
g_imu_yaw_deg
g_imu_ax_g
g_imu_ay_g
g_imu_az_g
g_imu_gx_dps
g_imu_gy_dps
g_imu_gz_dps
g_imu_packet_count
g_imu_checksum_error_count
```

### 7.5 判断是否正常

先看：

```text
g_board_app_loop_count
```

应持续增加。若不增加，检查 `while (1)` 是否调用了 `Board_App_Loop()`。

再看激光：

```text
g_laser_status
g_laser_module_id
g_laser_sample_count
g_laser_distance_mm
```

正常现象：

- `g_laser_module_id` 应接近或等于 `0xEACC`
- `g_laser_status` 为 `0` 时表示 `ATK_MS53L1M_OK`
- `g_laser_sample_count` 持续增加
- `g_laser_distance_mm` 随障碍物距离变化

再看 JY61P：

```text
g_imu_packet_count
g_imu_checksum_error_count
g_imu_roll_deg
g_imu_pitch_deg
g_imu_yaw_deg
```

正常现象：

- `g_imu_packet_count` 持续增加
- `g_imu_checksum_error_count` 不应快速连续增加
- 转动模块时，`g_imu_roll_deg`, `g_imu_pitch_deg`, `g_imu_yaw_deg` 应变化

### 7.6 常见异常判断

| 现象 | 优先检查 |
| --- | --- |
| `g_board_app_loop_count` 不增加 | `main.c` 是否调用 `Board_App_Loop()` |
| `g_laser_status = 2` | 激光 I2C 接线、供电、`PB8/PB9` remap、模块 ID |
| `g_laser_sample_count` 不增加 | 激光初始化是否成功，`g_laser_status` 是否长期非 0 |
| `g_imu_packet_count` 不增加 | `USART3` 是否启用、`PB10/PB11` 是否接反、JY61P 波特率是否为 115200、NVIC 是否开启 |
| `g_imu_checksum_error_count` 快速增加 | 串口波特率错误、线过长、接地不可靠、电源噪声 |
| 姿态角不变化但 packet 增加 | JY61P 输出内容配置异常，使用上位机恢复默认输出 |

## 8. 推荐的调试顺序

1. 先只确认 `g_board_app_loop_count` 增加
2. 再确认 `g_laser_module_id == 0xEACC`
3. 再确认 `g_laser_sample_count` 增加
4. 再确认 `g_imu_packet_count` 增加
5. 最后观察 `g_laser_distance_mm` 和 `g_imu_yaw_deg`

不要一开始就看控制效果。先确认两个传感器数据都稳定，再写小车控制逻辑。
