## 1. 系统架构总览

### 1.1 分层职责

系统按"决策层、控制层、执行层"划分。树莓派负责高层任务，STM32 负责实时采集、通信转发和安全保护，电机驱动与传感器负责底层执行和反馈。

| 层级 | 硬件 | 主要职责 | 关键输出 |
|---|---|---|---|
| 决策层 | 树莓派 4B | 视觉识别、路径规划、任务调度、比赛策略 | 速度指令、目标点、任务状态 |
| 控制层 | STM32F103C8T6 底盘控制板 | 传感器采样、命令校验、限幅、安全停车、电机通信分发 | 电机命令、状态上报、安全状态 |
| 感知层 | JY61P、ATK-MS53L1M | 姿态、角速度、加速度、前向或指定方向距离 | IMU 数据、距离数据 |
| 执行层 | ZDT_X42 电机驱动与麦克纳姆轮 | 底盘移动、闭环速度/位置执行 | 编码器/驱动状态、实际运动 |

### 1.2 当前物理链路

```text
树莓派 4B
  USB
   |
   v
CH340N USB 转串口
   |
   | 3.3V/TTL 串口逻辑，板级电平需实测确认
   v
STM32F103C8T6 USART2
  PA2 = USART2_TX
  PA3 = USART2_RX

STM32F103C8T6 I2C1 
  PB6 = I2C1_SCL
  PB7 = I2C1_SDA
   |---------------- JY61P IMU
   |---------------- ATK-MS53L1M / VL53L1X 激光测距

STM32F103C8T6 USART1
  PA9 = USART1_TX
  PA10 = USART1_RX
   |
   v
RS485模块 → ZDT_X42 电机通信链路
```

### 1.3 数据流向

| 方向 | 路径 | 数据内容 | 处理要求 |
|---|---|---|---|
| 上行反馈 | JY61P -> I2C1 -> STM32 -> USART2/CH340N -> 树莓派 | Roll、Pitch、Yaw、三轴加速度、三轴角速度、温度 | 带时间戳，保留最近一次有效值 |
| 上行反馈 | ATK-MS53L1M -> I2C1 -> STM32 -> USART2/CH340N -> 树莓派 | 距离 mm、测量状态、样本计数 | 距离异常时标记无效，不直接覆盖安全状态 |
| 上行反馈 | 电机驱动 -> USART3 -> STM32 -> USART2/CH340N -> 树莓派 | 在线状态、速度/位置反馈、错误码 | 按实际电机协议补全 |
| 下行控制 | 树莓派 -> USART2/CH340N -> STM32 -> USART3 -> 电机驱动 | 速度向量、目标位置、停止、急停解除、校准命令 | CRC 校验、序号检查、限幅、安全状态优先 |

## 2. STM32F103C8T6 引脚与外设分配

### 2.1 已确定引脚表

| STM32 引脚 | 外设/功能 | 连接对象 | 方向 | 状态 | 备注 |
|---|---|---|---|---|---|---|
| PB6 | I2C1_SCL | JY61P、ATK-MS53L1M | 双向 | 已确定 | 需在 CubeMX/HAL 中启用 I2C1 |
| PB7 | I2C1_SDA | JY61P、ATK-MS53L1M | 双向 | 已确定 | |
| PA9 | USART1_TX | RS485 模块 DI → ZDT_X42 电机 | STM32 输出 | 已确定 | 需另配 GPIO 控制 RS485 收发方向 |
| PA10 | USART1_RX | RS485 模块 RO ← ZDT_X42 电机 | STM32 输入 | 已确定 | 与电机通信模块共地 |
| PA2 | USART2_TX | CH340N RXD，树莓派通信 | STM32 输出 | 已确定 | 树莓派通过 USB 访问 CH340N 虚拟串口 |
| PA3 | USART2_RX | CH340N TXD，树莓派通信 | STM32 输入 | 已确定 | 需确认 CH340N TXD 电平安全 |
| PA13 | SWDIO | ST-LINK / 调试器 | 双向 | 保留 | 不作为业务 IO 使用 |
| PA14 | SWCLK | ST-LINK / 调试器 | 输入 | 保留 | 不作为业务 IO 使用 |
| NRST | 复位 | 调试器/复位电路 | 输入 | 保留 | 建议在调试接口引出 |
| PC13 | 状态 LED，可选 | 板载 LED | 输出 | 可选 | 低电平点亮的核心板常见连接，需按实际板验证 |

### 2.2 CubeMX / CubeIDE 配置要点

| 项目 | 建议配置 | 原因 |
|---|---|---|
| I2C1 | PB6(SCL)/PB7(SDA)，100 kHz 起步 | 默认引脚，不需remap；两个传感器共总线，先保证稳定 |
| USART2 | 异步串口，115200 bps 起步 | 树莓派和 STM32 主通信链路 |
| USART3 | 异步串口，按电机驱动协议设置 | 电机链路独立于树莓派链路 |
| SWD | 保持开启 | 便于 ST-LINK 在线调试 |
| GPIO 未用脚 | 统一配置为模拟输入或明确上下拉 | 降低悬空噪声和功耗 |

### 2.3 引脚占用原则

- PB6/PB7 归 I2C1 独占，由 JY61P 和 ATK-MS53L1M 共用。
- PA2/PA3 归树莓派通信链路独占。
- PA9/PA10 归电机 USART1 通信链路独占。
- PA13/PA14/NRST 保留给调试和下载。
- 后续新增外设时优先选择未占用的 GPIO，并同步更新本表。

## 3. 传感器连接与数据采集

### 3.1 I2C1 共总线设计

JY61P 和 ATK-MS53L1M 均接入 STM32 的 I2C1，总线引脚为 PB6/PB7。

| 设备 | 7-bit 地址 | HAL/8-bit 地址 | 主要数据 | 当前代码状态 |
|---|---|---|---|---|
| JY61P | 0x50 | 0xA0 | 加速度、角速度、Roll/Pitch/Yaw、温度 | .c/.h 已完成 |
| ATK-MS53L1M / VL53L1X | 0x29 | 0x52 | 距离 mm、测量状态 | .c/.h 已完成 |

I2C 电气要求：
- SCL/SDA 上拉必须接 5V（硬件板级确认）。
- 两个传感器和 STM32 必须共地。
- 调试阶段 I2C 速率先使用 100 kHz。
- 总线偶发卡死时固件需支持 I2C bus recovery。

### 3.2 JY61P 陀螺仪/IMU

#### 连接方式

| JY61P 引脚 | STM32 连接 | 说明 |
|---|---|---|
| VCC | 5V | 统一5V供电 |
| GND | GND | 共地 |
| SCL | PB6 / I2C1_SCL | I2C 时钟 |
| SDA | PB7 / I2C1_SDA | I2C 数据 |

| TX | 不连接 | 本版不使用串口模式 |

| 加速度 | AX=0x34 起 | raw / 32768 * 16g |
| 角速度 | GX=0x37 起 | raw / 32768 * 2000 dps |
| 欧拉角 | ROLL=0x3D 起 | raw / 32768 * 180 deg |
| 温度 | TEMP=0x40 | raw / 100 |

### 3.4 传感器使用建议

- IMU 适合用于航向角辅助、姿态异常检测和短时间运动稳定判断。
- 激光测距适合用于前向障碍、靠近货架/目标点时的距离判断。
- 两个传感器都在 I2C1 上，应避免阻塞式 I2C 操作影响另一个传感器。

## 4. 电机与树莓派通信

### 4.1 树莓派与 STM32 链路

树莓派通过 USB → CH340N → STM32 USART2(PA2/PA3)，波特率 115200 bps 起步。

建议下行命令帧格式：帧头 + 序号 + 消息类型 + 负载长度 + 负载 + CRC16。
建议上行状态帧格式：帧头 + 系统状态 + 故障位 + IMU摘要 + 激光摘要 + 电机摘要 + 心跳。

### 4.2 STM32 与电机驱动链路

| 项目 | 配置 |
|---|---|
| STM32 外设 | USART1 |
| 引脚 | PA9 = TX，PA10 = RX |
| 连接对象 | ZDT_X42 电机通信链路（经 RS485 转换） |
| 起步波特率 | 115200 bps |

## 5. 供电系统与功率预算

### 5.1 当前供电树

12V 输入 → 12V 母线（电机驱动）+ TPS5450 → +5V（树莓派、CH340N、STM32）+ TPS5450 → +7.5V（后续外设）。

### 5.2 功率预算（摘要）

| 负载 | 电压 | 功率估算 |
|---|---|---|
| 树莓派 4B | 5V | 峰值 15W |
| STM32F103 核心板 | 5V/3.3V | 小于 0.5W |
| JY61P | 3.3V/5V | 约 0.042W |
| ATK-MS53L1M | 3.3V/5V | 小于 0.15W |
| ZDT_X42 x4 | 12V | 待测 |

## 6. 数据接收与分发

### 6.1 STM32 内部数据模型

| 数据块 | 来源 | 更新周期 | 关键字段 |
|---|---|---|---|
| imu_state | JY61P/I2C1 | 10-20 ms | 姿态、角速度、加速度、时间戳、有效标志 |
| laser_state | ATK-MS53L1M/I2C1 | 100-200 ms | 距离、状态、时间戳、过近标志 |
| motor_state | ZDT_X42/USART3 | 20-100 ms | 在线状态、速度、位置、错误码 |
| cmd_state | 树莓派/USART2 | 每帧更新 | 速度目标、位置目标、模式、序号 |
| safety_state | STM32 本地状态机 | 1-5 ms | 急停、超时、限速、故障位 |
| link_state | USART2/USART3 | 每次收发更新 | 心跳时间、错误计数、丢包计数 |

### 6.2 推荐裸机调度节拍

## 7. 赛场运行状态与模块调用

### 7.1 状态机总览

POWER_ON → SELF_TEST → IDLE → READY → RUNNING（CRUISE / APPROACH_TARGET / AVOID_OBSTACLE / LOAD_OR_UNLOAD）→ FAULT（LINK_LOST / POSE_ABNORMAL / MOTOR_FAULT / E_STOP）→ RECOVERY

## 8. FreeRTOS 必要性评估

### 8.1 当前结论

当前阶段不强制使用 FreeRTOS。裸机超级循环 + 中断/DMA + 非阻塞状态机已可满足需求。

### 8.2 需要引入 FreeRTOS 的条件

- 树莓派下发高频速度命令且串口解析影响传感器采样。
- 需要持续读取四个电机实时状态。
- 激光、IMU、电机、树莓派、日志同时运行时主循环延迟不可控。
