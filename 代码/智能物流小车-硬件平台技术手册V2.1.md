## 1. 系统架构总览

### 1.1 分层职责

系统按“决策层、控制层、执行层”划分。树莓派负责高层任务，STM32 负责实时采集、通信转发和安全保护，电机驱动与传感器负责底层执行和反馈。

| 层级 | 硬件 | 主要职责 | 关键输出 |
|---|---|---|---|
| 决策层 | 树莓派 4B | 视觉识别、路径规划、任务调度、比赛策略 | 速度指令、目标点、任务状态 |
| 控制层 | STM32F103C8T6 底盘控制板 | 传感器采样、命令校验、限幅、安全停车、电机通信分发 | 电机命令、状态上报、安全状态 |
| 感知层 | JY61P、ATK-MS53L1M | 姿态、角速度、加速度、前向或指定方向距离 | IMU 数据、距离数据 |
| 执行层 | ZDT_X42 电机驱动与麦克纳姆轮 | 底盘移动、闭环速度/位置执行 | 编码器/驱动状态、实际运动 |

### 1.2 当前物理链路

```text
树莓派 4B
  USB
   |
   v
CH340N USB 转串口
   |
   | 3.3V/TTL 串口逻辑，板级电平需实测确认
   v
STM32F103C8T6 USART2
  PA2 = USART2_TX
  PA3 = USART2_RX

STM32F103C8T6 I2C1 
  PB6 = I2C1_SCL
  PB7 = I2C1_SDA
   |---------------- JY61P IMU
   |---------------- ATK-MS53L1M / VL53L1X 激光测距

STM32F103C8T6 USART1
  PA9 = USART1_TX
  PA10 = USART1_RX
   |
   v
RS485模块 → ZDT_X42 电机通信链路
```

### 1.3 数据流向

| 方向 | 路径 | 数据内容 | 处理要求 |
|---|---|---|---|
| 上行反馈 | JY61P -> I2C1 -> STM32 -> USART2/CH340N -> 树莓派 | Roll、Pitch、Yaw、三轴加速度、三轴角速度、温度 | 带时间戳，保留最近一次有效值 |
| 上行反馈 | ATK-MS53L1M -> I2C1 -> STM32 -> USART2/CH340N -> 树莓派 | 距离 mm、测量状态、样本计数 | 距离异常时标记无效，不直接覆盖安全状态 |
| 上行反馈 | 电机驱动 -> USART3 -> STM32 -> USART2/CH340N -> 树莓派 | 在线状态、速度/位置反馈、错误码 | 按实际电机协议补全 |
| 下行控制 | 树莓派 -> USART2/CH340N -> STM32 -> USART3 -> 电机驱动 | 速度向量、目标位置、停止、急停解除、校准命令 | CRC 校验、序号检查、限幅、安全状态优先 |

### 2.2 CubeMX / CubeIDE 配置要点

| 项目       | 建议配置                  | 原因                        |
| -------- | --------------------- | ------------------------- |
| I2C1     | PB6(SCL)/PB7(SDA)，100 kHz 起步    | 默认引脚，不需remap；两个传感器共总线，先保证稳定，再视波形提高速率   |
| USART2   | 异步串口，建议 115200 bps 起步 | 树莓派和 STM32 主通信链路，先以可靠调试为主 |
| USART3   | 异步串口，建议按电机驱动协议设置      | 电机链路独立于树莓派链路              |
| SWD      | 保持开启                  | 便于 ST-LINK 在线调试、变量观察和固件下载 |
| GPIO 未用脚 | 统一配置为模拟输入或明确上下拉       | 降低悬空噪声和功耗                 |

### 2.3 引脚占用原则

- `PB6/PB7` 归 I2C1 独占，由 JY61P 和 ATK-MS53L1M 共用。
- `PA2/PA3` 归树莓派通信链路独占。
- `PA9/PA10` 归电机 USART1 通信链路独占，经 RS485 模块连接 ZDT_X42。
- `PA13/PA14/NRST` 保留给调试和下载，不参与普通 IO 扩展。
- 后续新增急停、蜂鸣器、状态灯、电池 ADC 时，应优先选择未占用且方便布线的 GPIO，并同步更新本表。

## 3. 传感器连接与数据采集

### 3.1 I2C1 共总线设计

JY61P 和 ATK-MS53L1M 当前均接入 STM32 的 I2C1，总线引脚为 `PB8/PB9`。两个设备地址不同，不存在地址冲突。

| 设备 | 7-bit 地址 | HAL/8-bit 地址 | 主要数据 | 当前代码状态 |
|---|---:|---:|---|---|
| JY61P | `0x50` | `0xA0` | 加速度、角速度、Roll/Pitch/Yaw、温度 | `.c/.h` 已完成，使用 I2C 寄存器读取 |
| ATK-MS53L1M / VL53L1X | `0x29` | `0x52` | 距离 mm、测量状态 | `.c/.h` 已完成，使用 VL53L1X 官方 API 适配 |

I2C 电气要求：

- SCL/SDA 上拉必须接 5V（硬件板级确认）。传感器模块由 5V 供电，I2C 上拉已确认为 5V。
- 两个传感器和 STM32 必须共地。
- 调试阶段 I2C 速率先使用 100 kHz。线长较短、波形干净后可评估 400 kHz。
- 如果总线偶发卡死，固件需要支持 I2C bus recovery：将 SCL 临时配置为 GPIO 输出，产生 9 个时钟后重新初始化 I2C。

### 3.2 JY61P 陀螺仪/IMU

#### 连接方式

| JY61P 引脚 | STM32 连接       | 说明                    |
| -------- | -------------- | --------------------- |
| VCC      | 3.3V 或 5V      | 推荐优先 3.3V，避免 I2C 电平风险 |
| GND      | GND            | 与 STM32 和电机通信地共地      |
| SCL      | PB7 / I2C1_SCL | I2C 时钟                |
| SDA      | PB6 / I2C1_SDA | I2C 数据                |
| RX       | 不连接            | 本版不使用 JY61P 串口模式      |
| TX       | 不连接            | 本版不使用 JY61P 串口模式      |

#### 电气与功耗

| 参数 | 当前文档取值 | 备注 |
|---|---:|---|
| 工作电压 | 3.3V-5V | 以模块实物规格为准 |
| 工作电流 | 约 8.4 mA @ 5V | 约 0.042 W；若 3.3V 供电，功耗约 0.028 W 量级 |
| 数据读取方式 | I2C 寄存器读取 | 代码中连续读取 `0x34` 起始的数据寄存器 |

#### 代码接口

当前代码中 `JY61P_ReadSensor()` 从寄存器 `0x34` 起读取 26 字节，并转换为以下物理量：

| 数据 | 寄存器 | 转换关系 |
|---|---|---|
| 加速度 | `AX=0x34` 起 | `raw / 32768 * 16g` |
| 角速度 | `GX=0x37` 起 | `raw / 32768 * 2000 dps` |
| 欧拉角 | `ROLL=0x3D` 起 | `raw / 32768 * 180 deg` |
| 温度 | `TEMP=0x40` | `raw / 100` |

建议应用层保留以下数据：

- `imu.roll_deg`
- `imu.pitch_deg`
- `imu.yaw_deg`
- `imu.ax_g / imu.ay_g / imu.az_g`
- `imu.gx_dps / imu.gy_dps / imu.gz_dps`
- `imu.last_update_ms`
- `imu.valid`
- `imu.error_count`

### 3.3 ATK-MS53L1M 激光测距模块

#### 连接方式

| ATK-MS53L1M 引脚 | STM32 连接       | 说明                         |
| -------------- | -------------- | -------------------------- |
| VCC            | 3.3V 或 5V      | 推荐优先 3.3V，或确认模块 I2C 已做电平转换 |
| GND            | GND            | 与 STM32 共地                 |
| SCL            | PB6 / I2C1_SCL | 与 JY61P 共用                 |
| SDA            | PB7 / I2C1_SDA | 与 JY61P 共用                 |
| TXD            | 不连接            | I2C 模式不使用                  |
| RXD            | 不连接            | I2C 模式不使用                  |

#### 电气与功耗

| 参数 | 当前文档取值 | 备注 |
|---|---:|---|
| 工作电压 | 3.3V-5V | 模块供电可宽压，但 I2C 电平仍按 3.3V 管理 |
| 工作电流 | 小于 30 mA | 5V 下小于 0.15 W，3.3V 下小于 0.10 W |
| 测量距离 | 40 mm-4000 mm | 长距离模式取决于环境反射率 |
| 激光安全 | 940 nm，Class 1 | 正常使用下人眼安全 |

#### 代码接口

当前 `ATK_MS53L1M_Init()` 使用 HAL I2C 句柄初始化 VL53L1X，并设置：

| 项目 | 当前代码值 | 说明 |
|---|---:|---|
| 模块 ID 寄存器 | `0x010F` | 读取后应为 `0xEACC` |
| 测距模式 | Long | 适合较长距离检测 |
| Timing budget | 140000 us | 单次测量预算 |
| Inter-measurement | 150 ms | 当前有效采样率约 6-7 Hz |
| 应用层轮询周期 | 20 ms | 轮询是否有新数据，不代表每 20 ms 都有新距离 |

建议应用层保留以下数据：

- `laser.distance_mm`
- `laser.valid`
- `laser.last_update_ms`
- `laser.sample_count`
- `laser.status`
- `laser.too_close`

### 3.4 传感器使用建议

- IMU 适合用于航向角辅助、姿态异常检测和短时间运动稳定判断，不应单独承担长距离定位。
- 激光测距适合用于前向障碍、靠近货架/目标点时的距离判断。单点激光存在视场限制，比赛中应结合视觉或多角度传感器判断。
- 两个传感器都在 I2C1 上，任何阻塞式 I2C 操作都会影响另一个传感器。固件中应避免长时间等待，错误后快速返回并上报状态。

## 4. 电机与树莓派通信

### 4.1 树莓派与 STM32 链路

树莓派通过 USB 连接控制板上的 CH340N，CH340N 再连接 STM32 的 USART2。

| 项目 | 配置 |
|---|---|
| STM32 外设 | USART2 |
| STM32 引脚 | PA2 = TX，PA3 = RX |
| 物理桥接 | CH340N USB 转串口 |
| 树莓派侧设备名 | 通常为 `/dev/ttyUSB0`，具体以 `dmesg` 或 `ls /dev/ttyUSB*` 为准 |
| 起步波特率 | 115200 bps |
| 推荐协议 | 自定义二进制帧，带长度、序号、类型、CRC16 |

建议下行命令帧至少包含：

| 字段 | 说明 |
|---|---|
| frame_head | 固定帧头，用于同步 |
| seq | 序号，用于丢包和乱序检测 |
| msg_type | 速度控制、位置控制、停止、校准、心跳等 |
| payload_len | 负载长度 |
| payload | 速度向量、目标点、参数 |
| crc16 | 完整性校验 |

建议上行状态帧至少包含：

| 字段 | 说明 |
|---|---|
| system_state | 当前运行状态 |
| fault_flags | 故障位图 |
| imu | 姿态、角速度、加速度摘要 |
| laser | 距离和有效标志 |
| motor | 电机在线状态、速度/位置摘要 |
| heartbeat | STM32 心跳计数 |

### 4.2 CH340N 注意事项

硬件审查中识别到板载 USB 串口为 CH340N。CH340N 属于内置时钟版本，不需要外部 12 MHz 晶振。后续硬件文档不要再按需要外部晶振的 CH340 型号描述。

需要继续验证的风险：

- 如果 CH340N 由 5V 供电，需确认其 TXD 到 STM32 RX 的电平是否安全。
- 若 USB VBUS 未参与检测，而 CH340N 由板上 5V 自供电，可能存在 USB 自供电设备规范风险。比赛功能上可能可用，但正式板应评估 VBUS 检测或供电隔离。
- 树莓派和 STM32 之间的串口协议必须有心跳和超时停车，不能依赖 USB 连接一直可靠。

### 4.3 STM32 与电机驱动链路

| 项目 | 配置 |
|---|---|
| STM32 外设 | USART1 |
| STM32 引脚 | PA9 = TX，PA10 = RX |
| 连接对象 | ZDT_X42 电机通信链路 |
| 起步波特率 | 按电机驱动设置，建议先统一 115200 bps |
| 协议 | 以 ZDT_X42 当前配置为准，若使用 Modbus-RTU 则需 CRC16 和从机地址 |

本版文档按“USART1 是 MCU 侧电机串口”记录。电机链路使用 RS485 转换模块，USART1 负责 TTL 串口收发，外部还需要：

- RS485 收发器或 ZDT_RS485 模块。
- A/B 差分线正确连接。
- 终端电阻按总线长度决定。
- 收发方向控制引脚，如果收发器不是自动方向控制，需要在 STM32 上另分配 GPIO。

建议电机地址约定：

| 地址 | 位置 | 备注 |
|---:|---|---|
| `0x01` | 左前轮 | 待实机确认 |
| `0x02` | 右前轮 | 待实机确认 |
| `0x03` | 左后轮 | 待实机确认 |
| `0x04` | 右后轮 | 待实机确认 |

### 4.4 安全通信策略

| 风险 | STM32 处理策略 |
|---|---|
| 树莓派心跳超时 | 500 ms-1000 ms 内进入安全停车状态 |
| 下行帧 CRC 错误 | 丢弃该帧，错误计数加 1，不改变运动目标 |
| 下行速度过大 | 限幅后执行，并上报限幅标志 |
| 电机链路无响应 | 停止对应电机或全部停车，按故障等级处理 |
| 传感器异常 | 标记数据无效，禁止使用异常值触发运动增益 |

## 5. 供电系统与功率预算

### 5.1 当前供电树

硬件审查报告显示，当前控制板是 12V 输入母线加两路降压输出，不是由降压模块生成稳压 12V。

```text
12V 电池/适配器输入
  |
  +-- 12V 母线 -> 电机驱动、大功率负载
  |
  +-- TPS5450 -> +5V
  |      +-- 树莓派 4B
  |      +-- CH340N
  |      +-- STM32 核心板 5V 输入或其它 5V 外设
  |
  +-- TPS5450 -> +7.5V
         +-- 后续外设/机械臂相关接口，按实际连接确认
```

I2C 传感器可以使用 3.3V 或 5V 供电，但 SCL/SDA 必须维持 3.3V 逻辑。实际接线建议优先让传感器跟 STM32 使用同一 3.3V 逻辑域，降低总线电平风险。

### 5.2 功率预算

| 负载 | 电压 | 电流估算 | 功率估算 | 备注 |
|---|---:|---:|---:|---|
| 树莓派 4B | 5V | 峰值按 3A 预留 | 15W | 摄像头、USB 外设和算法负载会抬高峰值 |
| CH340N | 5V | 约 10-30 mA，待实测 | 小于 0.15W | 以实际芯片和 USB 状态为准 |
| STM32F103 核心板 | 5V 输入或 3.3V | 数十 mA 量级 | 小于 0.5W | 包含核心板 LDO 损耗 |
| JY61P | 3.3V 或 5V | 约 8.4 mA @ 5V | 约 0.042W @ 5V | 建议确认 I2C 上拉电压 |
| ATK-MS53L1M | 3.3V 或 5V | 小于 30 mA | 小于 0.15W @ 5V | VL53L1X 测距模块 |
| ZDT_X42 电机驱动 x4 | 12V | 按驱动电流设置确认 | 待测 | 启停瞬态可能远高于静态估算 |
| 摄像头 | 5V/USB | 按型号确认 | 待测 | 由树莓派 USB 口或独立电源供电 |

### 5.3 电源完整性要求

- 12V 输入端应有保险丝、开关、反接保护和足够的输入储能电容。
- 电机急启急停会导致 12V 母线跌落和噪声，需用示波器观察 5V、3.3V 轨。
- STM32、传感器、CH340N、电机通信模块必须共地，但大电流电机回流不要穿过传感器地线。
- 5V 给树莓派供电时需保证线径和接插件压降，避免树莓派低压降频或重启。
- I2C 传感器附近建议放置 `100nF + 10uF` 去耦。

### 5.4 硬件审查中建议保留的风险项

| 风险项 | 影响 | 建议 |
|---|---|---|
| +5V 实测/计算约 5.08V | 通常可用，但命名和目标值需统一 | BOM 和原理图统一标注目标输出 |
| TPS5450 BOOT 电容取值需核对 | 可能影响高边驱动稳定性 | 按数据手册复核 |
| 输入保险丝规格描述不一致 | 保护策略不清晰 | 明确保持电流、熔断电流、额定电压 |
| USB 自供电/VBUS 检测 | USB 合规和反灌风险 | 正式板增加检测或隔离 |
| CH340N 与 STM32 电平 | MCU 输入安全风险 | 优先确认 CH340N TXD 电平是否安全 |

## 6. 数据接收与分发

### 6.1 STM32 内部数据模型

建议 STM32 内部维护以下缓存结构。具体 C 结构体可后续在固件文档中定义。

| 数据块 | 来源 | 更新周期建议 | 关键字段 |
|---|---|---:|---|
| `imu_state` | JY61P/I2C1 | 10-20 ms | 姿态、角速度、加速度、时间戳、有效标志 |
| `laser_state` | ATK-MS53L1M/I2C1 | 100-200 ms | 距离、状态、时间戳、过近标志 |
| `motor_state` | ZDT_X42/USART3 | 20-100 ms | 在线状态、速度、位置、错误码 |
| `cmd_state` | 树莓派/USART2 | 每帧更新 | 速度目标、位置目标、模式、序号 |
| `safety_state` | STM32 本地状态机 | 1-5 ms | 急停、超时、限速、故障位 |
| `link_state` | USART2/USART3 | 每次收发更新 | 心跳时间、错误计数、丢包计数 |

### 6.2 下行命令处理

```text
USART2 RX 中断或 DMA
  -> ring buffer
  -> 帧同步
  -> 长度检查
  -> CRC 检查
  -> 序号/时间戳检查
  -> 命令解析
  -> 安全限幅
  -> 更新 cmd_state
  -> 电机命令生成
  -> USART3 发送
```

处理原则：

- 任何 CRC 错误帧都不允许改变当前运动目标。
- 停止、急停类命令优先级高于普通速度命令。
- 树莓派超时后，STM32 应覆盖运动目标为零速，并进入安全停车状态。
- 校准类命令需要在小车静止状态下执行，执行期间拒绝运动命令。

### 6.3 上行数据分发

```text
I2C1 传感器采样
USART3 电机状态采样
本地安全状态机
  -> 统一打包 telemetry
  -> USART2/CH340N
  -> 树莓派任务调度/日志/可视化
```

上行包建议分两类：

| 类型 | 周期 | 内容 |
|---|---:|---|
| 快速状态包 | 20-50 ms | 系统状态、故障位、IMU 摘要、电机在线摘要 |
| 慢速诊断包 | 200-1000 ms | 传感器计数、错误计数、电源状态、调试变量 |

### 6.4 推荐裸机调度节拍

在不使用 FreeRTOS 的阶段，建议采用“超级循环 + 中断/DMA + 非阻塞状态机”：

| 模块 | 触发方式 | 建议周期 | 说明 |
|---|---|---:|---|
| USART2 接收 | 中断或 DMA | 实时 | 只入队，不在中断中解析复杂协议 |
| USART3 接收 | 中断或 DMA | 实时 | 电机响应帧入队 |
| 协议解析 | 主循环 | 1-5 ms | 每次处理有限字节，避免长时间占用 |
| 安全状态机 | 主循环或定时器 | 1-5 ms | 检查心跳、急停、传感器异常 |
| IMU 读取 | 主循环 | 10-20 ms | I2C 读取失败时快速返回 |
| 激光读取 | 主循环 | 100-200 ms | 当前代码 150 ms 测量周期 |
| 电机命令刷新 | 主循环 | 10-20 ms | 只在目标变化或周期到达时发送 |
| 状态上报 | 主循环 | 20-100 ms | 快慢包分开 |

## 7. 赛场运行状态与模块调用

### 7.1 状态机总览

```text
POWER_ON
  -> SELF_TEST
  -> IDLE
  -> READY
  -> RUNNING
       |-> CRUISE
       |-> APPROACH_TARGET
       |-> AVOID_OBSTACLE
       |-> LOAD_OR_UNLOAD
  -> FAULT
       |-> LINK_LOST
       |-> POSE_ABNORMAL
       |-> MOTOR_FAULT
       |-> E_STOP
  -> RECOVERY
```

### 7.2 各场景模块调用

| 场景 | 触发条件 | 树莓派模块 | STM32 模块 | 电机状态 | 传感器状态 |
|---|---|---|---|---|---|
| 上电自检 | 电源打开或复位 | 等待串口枚举，启动底盘节点 | 初始化 GPIO、I2C1、USART2、USART3、传感器、电机链路 | 禁止运动 | 读取 ID 和首帧数据 |
| 待命 | 自检通过，无任务 | 保持心跳，等待任务 | 周期上报状态，电机目标为零 | 停止保持 | 持续采样 |
| 巡航 | 收到路径速度命令 | 视觉/路径规划输出速度 | 校验命令、限幅、生成电机命令 | 按目标运动 | IMU 和距离持续上报 |
| 靠近目标 | 视觉识别到货架/目标点 | 降低速度，输出精细调整命令 | 增加距离判断，限制最大速度 | 低速运行 | 激光距离参与判断 |
| 搬运/装卸 | 到达任务点 | 控制机械臂或等待外部动作 | 底盘锁定或低速微调 | 停止或微动 | IMU 检查姿态稳定 |
| 避障 | 距离小于阈值或视觉报警 | 重新规划或下发停止 | 立即减速/停车，上报障碍标志 | 减速或停止 | 激光过近标志置位 |
| 姿态异常 | Roll/Pitch/角速度超过阈值 | 降级策略或报警 | 限速、停车、上报故障 | 停止或低速 | IMU 异常标志置位 |
| 通信丢失 | USART2 心跳超时 | 串口重连 | 自主停车，保持故障状态 | 停止 | 继续本地采样 |
| 急停/严重故障 | 急停输入、电机严重错误 | 显示故障并等待人工处理 | 立即停车，锁定故障态 | 禁止输出运动 | 状态继续上报 |

### 7.3 建议阈值管理

以下阈值先作为调试默认值，正式比赛前必须通过实测调整：

| 阈值 | 默认建议 | 说明 |
|---|---:|---|
| 树莓派心跳超时 | 500 ms | 超过则 STM32 自主停车 |
| 普通状态上报周期 | 50 ms | 兼顾实时性和串口带宽 |
| 激光避障预警距离 | 300-500 mm | 按车速和制动距离调整 |
| 激光硬停车距离 | 100-200 mm | 按传感器安装位置调整 |
| 姿态异常 Roll/Pitch | 8-12 deg | 防止倾倒或严重碰撞后继续运行 |
| 电机命令刷新 | 20 ms | 50 Hz 控制频率起步 |

## 8. FreeRTOS 必要性评估

### 8.1 当前结论

当前阶段不强制必须使用 FreeRTOS。以 STM32F103C8T6 的资源和当前功能量看，裸机超级循环配合串口中断/DMA、I2C 非阻塞式轮询、定时状态机，已经可以完成：

- JY61P 和 ATK-MS53L1M 数据轮询。
- USART2 树莓派通信。
- USART3 电机通信。
- 心跳超时与安全停车。
- 基础状态上报。

需要注意：低延迟响应主要依赖中断、DMA、合理的状态机和短临界区，不是简单引入 RTOS 就能自动获得。若任务优先级、栈、锁和阻塞等待设计不当，FreeRTOS 反而会增加调试复杂度。

### 8.2 建议采用的阶段策略

| 阶段 | 建议 | 原因 |
|---|---|---|
| P0 联调 | 裸机超级循环 + 中断/DMA | 最少变量，便于确认硬件、引脚和协议 |
| P1 稳定化 | 继续裸机，补齐状态机和错误恢复 | 当前任务量仍可控 |
| P2 多模块并发 | 评估轻量 FreeRTOS | 当通信、传感器、日志、安全逻辑互相阻塞时再引入 |
| P3 比赛优化 | 视调试结果决定 | 如果裸机延迟稳定，不必为了形式使用 RTOS |

### 8.3 需要引入 FreeRTOS 的条件

满足以下任意两项时，建议迁移到轻量 FreeRTOS：

- 树莓派下发高频速度命令，且串口解析影响传感器采样。
- 需要持续读取四个电机的实时状态，USART3 收发占用明显增加。
- 激光、IMU、电机、树莓派、日志同时运行时，主循环最长执行时间不可控。
- 急停、通信超时、电机故障要求独立高优先级处理。
- 后续新增机械臂、更多传感器或显示屏，模块之间开始争用 CPU。

### 8.4 如果使用 FreeRTOS 的任务草案

STM32F103C8T6 RAM 只有 20 KB，任务数量和栈必须严格控制。建议只使用静态创建，禁用不必要的动态分配和大缓冲。

| 任务 | 优先级 | 周期 | 职责 | 栈建议 |
|---|---:|---:|---|---:|
| SafetyTask | 最高 | 1-5 ms | 急停、心跳超时、故障锁定 | 128-192 words |
| CommsTask | 高 | 5-10 ms | USART2 协议解析、状态上报触发 | 192-256 words |
| MotorTask | 中高 | 10-20 ms | 电机命令刷新、状态读取 | 192-256 words |
| SensorTask | 中 | 10-20 ms | IMU 读取、激光状态轮询 | 256 words |
| TelemetryTask | 低 | 50-200 ms | 慢速诊断、日志、调试输出 | 192-256 words |

FreeRTOS 资源约束：

- `heap` 建议从 4 KB 起步，实测后再调整。
- 每个任务必须用 `uxTaskGetStackHighWaterMark()` 验证栈余量。
- 禁止在多个任务中直接调用阻塞式 printf。
- I2C 和串口发送必须有互斥或单一所有者任务，避免并发访问外设句柄。
- 安全停车逻辑不能依赖低优先级任务。

### 8.5 低延迟实现重点

无论是否使用 FreeRTOS，都应做到：

- USART2、USART3 接收使用中断或 DMA 环形缓冲。
- 急停输入使用外部中断或高频安全状态机。
- I2C 读写设置短超时，失败后记录错误并返回。
- 主循环或任务中不做长时间 `HAL_Delay()`。
- 所有运动命令都有时间戳，旧命令自动失效。
- 看门狗在系统卡死时强制复位，复位后默认电机停止。

## 9. 优化建议与后续待确认项

### 9.1 近期优先优化

| 优先级 | 项目 | 建议动作 |
|---|---|---|
| P0 | 确认电机物理层 | 确认 USART1 后面 RS485 模块的接线和方向控制 GPIO；若为 RS485，补充方向控制引脚 |
| P0 | 树莓派通信协议 | 定义二进制帧格式、CRC16、心跳、超时停车逻辑 |
| P0 | I2C 电平 | 实测 PB6/PB7 上拉电压，确保不是 5V |
| P0 | 电机地址和参数 | 记录四个 ZDT_X42 地址、细分、工作模式、限速、加速度 |
| P1 | 电源纹波 | 示波器测 12V、5V、3.3V 在电机急启急停时的波形 |
| P1 | CH340N 电平 | 实测 CH340N TXD 高电平，确认 STM32 输入安全 |
| P1 | 急停输入 | 确认是否已有急停按钮、接法和对应 GPIO |
| P1 | 传感器安装 | 记录 IMU 坐标系、激光安装方向和离地高度 |
| P2 | 电池监测 | 增加 12V 母线 ADC 分压或外置电量模块 |
| P2 | FreeRTOS 迁移 | 当裸机主循环延迟不可控时再启动 |

### 9.2 固件优化建议

- 为 USART2 和 USART3 分别实现 ring buffer，解析逻辑放在主循环或任务中。
- 所有上行数据带 `last_update_ms`，树莓派端不要使用过期传感器值。
- 运动命令统一经过限幅层，限制最大速度、最大角速度、最大加速度。
- 电机停止命令要有“普通减速停止”和“立即停止”两类接口。
- 使用 IWDG 独立看门狗，喂狗点放在主循环完整运行之后。
- 保留 ST-LINK Live Expressions 调试变量，便于无串口输出时验证传感器。

### 9.3 硬件优化建议

- I2C 线较长时可串联 22-100 ohm 小电阻抑制边沿振铃，并用示波器确认上升沿。
- 传感器线束与电机电源线分开走线，避免大电流噪声串入 I2C。
- 电机电源端增加足够储能电容，并确认保险丝和线束电流余量。
- Type-C/USB 口建议增加低电容 ESD 保护。
- 正式 PCB 上将 SWDIO、SWCLK、NRST、3.3V、GND 引出为稳定调试接口。
